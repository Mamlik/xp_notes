Выпуклая Оболочка (Convex Hull)

Алгоритмы Грэма, Джарвиса, вращающиеся суппорты

Постановка задачи. Дано множество P = {p₁, p₂, …, pₙ} точек на плоскости ℝ². Требуется найти выпуклую оболочку conv(P) — наименьшее выпуклое множество, содержащее все точки P.

📐 Определение — Выпуклая оболочка

$$\text{conv}(P) = \left\{ \sum_i \lambda_i p_i \;\middle|\; p_i \in P,\; \lambda_i \geq 0,\; \sum_i \lambda_i = 1 \right\}$$

Эквивалентно: наименьший выпуклый многоугольник Q, такой что P ⊆ Q.

Ключевое свойство: точка q ∈ conv(P) тогда и только тогда, когда она является выпуклой комбинацией точек из P.

1.1 Алгоритм Грэма (Graham Scan)

Основная идея. Выбираем опорную точку p₀ с наименьшей координатой Y (при равенстве — с наименьшей X). Все остальные точки сортируем по полярному углу относительно p₀. Затем проходим по отсортированным точкам, поддерживая стек, в котором каждые три последовательные точки образуют левый поворот.

📐 Определение — Векторное произведение (cross product)

$$\text{cross}(O, A, B) = (A_x - O_x)(B_y - O_y) - (A_y - O_y)(B_x - O_x)$$

Геометрический смысл:

cross > 0 → поворот O → A → B левый (CCW)

cross < 0 → поворот O → A → B правый (CW)

cross = 0 → точки O, A, B коллинеарны

Python
def cross(O, A, B):
    return (A[0]-O[0])*(B[1]-O[1]) - (A[1]-O[1])*(B[0]-O[0])

def convex_hull_graham(pts):
    pts = sorted(set(map(tuple, pts)))
    n = len(pts)
    if n <= 1: return pts
    # Нижняя цепь: движемся слева направо
    lower = []
    for p in pts:
        # Пока последние три не образуют левый поворот — убираем вершину
        while len(lower) >= 2 and cross(lower[-2], lower[-1], p) <= 0:
            lower.pop()
        lower.append(p)
    # Верхняя цепь: движемся справа налево
    upper = []
    for p in reversed(pts):
        while len(upper) >= 2 and cross(upper[-2], upper[-1], p) <= 0:
            upper.pop()
        upper.append(p)
    # Объединяем (исключаем дублирование крайних точек)
    return lower[:-1] + upper[:-1]
✓ Доказательство — Корректность алгоритма Грэма

Инвариант цикла: стек lower содержит вершины выпуклой цепи (только левые повороты).

База: при |lower| ≤ 1 инвариант тривиален.

Переход: при добавлении точки p проверяем cross(lower[-2], lower[-1], p). Если cross ≤ 0, то lower[-1] не может быть на нижней оболочке, так как p расположена правее или на линии lower[-2] → lower[-1]. Удаляем lower[-1]. Повторяем до выполнения условия.

Завершение: когда все точки обработаны, стек содержит нижнюю выпуклую цепь от pts[0] до pts[-1]. Аналогично для верхней. Объединение lower[:-1] + upper[:-1] даёт CCW-обход оболочки. ∎

✓ Доказательство — Сложность O(n log n)

1. Сортировка n точек: O(n log n).

2. Построение lower + upper: каждая точка добавляется в стек ровно 1 раз и удаляется не более 1 раза. Итого O(n) для обоих проходов.

3. Доминирует сортировка: T(n) = O(n log n).

Нижняя оценка: задача сортировки сводится к Convex Hull (точки (x, x²) лежат на параболе, оболочка даёт их порядок). Следовательно, Ω(n log n) — оптимально.

1.2 Алгоритм Джарвиса (Jarvis March / Gift Wrapping)

Идея. Начинаем с самой левой (крайней) точки. На каждом шаге ищем точку, которая является самой «правой» относительно текущего направления: т.е. точку q, такую что все остальные точки лежат слева от луча p → q. Алгоритм работает без предварительной сортировки, но на каждый шаг тратит O(n).

Python
def jarvis_march(pts):
    n = len(pts)
    if n < 3: return pts
    hull = []
    # Стартуем с крайне левой точки (минимальный X)
    start = min(range(n), key=lambda i: (pts[i][0], pts[i][1]))
    cur = start
    while True:
        hull.append(pts[cur])
        # next: точка, для которой все остальные — слева
        nxt = (cur + 1) % n
        for r in range(n):
            # cross < 0: r правее cur→nxt => обновляем nxt
            if cross(pts[cur], pts[nxt], pts[r]) < 0:
                nxt = r
        cur = nxt
        if cur == start:  # завершили обход
            break
    return hull
✓ Доказательство — Корректность Jarvis March

На каждом шаге из точки p мы выбираем q, минимизирующую угол поворота.

Утверждение: после полного обхода hull содержит все вершины оболочки.

Если q* — следующая вершина оболочки CCW, то для любой другой точки r: cross(p, q*, r) ≥ 0 (r левее или на луче p→q*). Следовательно, алгоритм выберет именно q*.

Цикл завершается, когда мы вернулись в start — что происходит ровно после обхода всех h вершин оболочки. Следовательно, hull корректен. ∎

1.3 Вращающиеся суппорты (Rotating Calipers)

Метод вращающихся суппортов (Shamos, 1978) позволяет решать задачи на выпуклых многоугольниках за O(n) после построения оболочки. Суппорты — пара параллельных прямых, касающихся оболочки. Поворачивая их, мы перебираем все антиподальные пары точек.

📐 Теорема — Диаметр через суппорты

Диаметр выпуклого n-угольника достигается на антиподальной паре.

Алгоритм вращающихся суппортов за O(n) обходит все антиподальные пары:

1. Найти вершину, максимально удалённую от ребра hull[0] → hull[1].

2. Синхронно двигать оба указателя i и j по оболочке: если area(hull[i], hull[i+1], hull[j+1]) > area(hull[i], hull[i+1], hull[j]), то j = (j+1) % n, иначе i = (i+1) % n.

3. На каждом шаге обновлять максимальное расстояние.

Корректность: все антиподальные пары будут рассмотрены, так как при монотонном повороте суппортов j строго возрастает между последовательными увеличениями i. ∎

Python
import math

def dist2(A, B):  # квадрат расстояния
    return (A[0]-B[0])**2 + (A[1]-B[1])**2

def polygon_diameter(hull):
    n = len(hull)
    if n == 1: return 0
    if n == 2: return math.sqrt(dist2(hull[0], hull[1]))
    # Ищем начальное j — максимально удалённое от hull[0]→hull[1]
    j = 1
    while abs(cross(hull[0], hull[1], hull[(j+1)%n])) > \
          abs(cross(hull[0], hull[1], hull[j])):
        j += 1
    best = 0
    i = 0
    while i != j:  # пока суппорты не совпали
        best = max(best, dist2(hull[i], hull[j]))
        c1 = cross(hull[i], hull[(i+1)%n], hull[(j+1)%n])
        c2 = cross(hull[i], hull[(i+1)%n], hull[j])
        if c1 > c2:
            j = (j+1) % n
        else:
            i = (i+1) % n
    return math.sqrt(best)

# Применения вращающихся суппортов:
# - Диаметр множества точек: O(n log n) с построением оболочки
# - Минимальный описывающий прямоугольник: O(n)
# - Ширина выпуклого многоугольника: O(n)
# - Расстояние между двумя выпуклыми многоугольниками: O(n+m)
💡 Замечание — Алгоритм Чана (Chan's Algorithm, 1996)

Объединяет Graham и Jarvis за O(n log h), где h — размер оболочки.

Идея: угадываем H = h (параметр), делаем ⌈n/H⌉ групп по H точек, строим оболочки групп алгоритмом Грэма за O(n log H), затем запускаем Jarvis на этих оболочках (H шагов по O(n/H · log H)).

Если H ≥ h — успех; иначе удваиваем H и повторяем. Суммарно: O(n log h) — оптимальный алгоритм для задачи.

Convex Hull Trick (CHT)

Оптимизация ДП · Дерево Ли Чао

Convex Hull Trick — техника оптимизации динамического программирования, применимая когда переходы имеют вид:

$$\text{dp}[i] = \min_{j < i} \{ \text{dp}[j] + m[j] \cdot x[i] + c[j] \}$$

где m[j], x[i] — монотонные или произвольные числа.

Геометрическая интерпретация. Каждый переход j задаёт прямую l_j(t) = m[j]·t + b[j], где b[j] = dp[j] + c[j]. При фиксированном i нас интересует минимальное значение прямых в точке t = x[i]. Множество прямых, образующих нижнюю огибающую — это в точности выпуклая оболочка в двойственном пространстве. Отсюда и название.

2.1 Геометрическое доказательство
📐 Теорема — Нижняя огибающая прямых выпукла

Пусть дан набор прямых {l_j(x) = m_j·x + b_j}. f(x) = min_j l_j(x) — их нижняя огибающая.

Утверждение: f(x) — выпуклая функция.

Доказательство: покажем, что f(λx + (1-λ)y) ≤ λf(x) + (1-λ)f(y). Пусть j* = argmin_j l_j(λx + (1-λ)y). Тогда:

$$f(\lambda x + (1-\lambda)y) = l_{j^*}(\lambda x + (1-\lambda)y) = \lambda l_{j^*}(x) + (1-\lambda) l_{j^*}(y)$$

Так как l_{j*}(x) ≥ f(x) и l_{j*}(y) ≥ f(y), получаем:

$$f(\lambda x + (1-\lambda)y) \leq \lambda f(x) + (1-\lambda) f(y)$$

Следствие: для минимизации оптимальные прямые упорядочены по x. ∎

2.2 Критерий удаления прямой (функция bad)

При добавлении прямых с убывающими наклонами m₁ ≥ m₂ ≥ m₃ средняя прямая l₂ «бесполезна» (никогда не является минимумом), если пересечение l₁ и l₃ лежит левее пересечения l₁ и l₂.

✓ Доказательство — Формула функции bad(l1, l2, l3)

Пересечение l₁ и l₂: x₁₂ = (b₂-b₁)/(m₁-m₂)

Пересечение l₁ и l₃: x₁₃ = (b₃-b₁)/(m₁-m₃)

l₂ никогда не оптимальна ⟺ x₁₃ ≤ x₁₂

Так как m₁ > m₂ > m₃, оба знаменателя > 0. Умножаем:

$$(b_3 - b_1)(m_1 - m_2) \leq (b_2 - b_1)(m_1 - m_3)$$

Это и есть проверка bad(l1, l2, l3). ∎

Python
# ── Статический CHT: наклоны убывают, запросы возрастают ──
class CHT_monotone:
    def __init__(self):
        self.lines = []  # (m, b), m убывает
        self.ptr = 0     # указатель для монотонных запросов

    def bad(self, l1, l2, l3):
        # l2 никогда не оптимальна при наличии l1 и l3
        return (l3[1]-l1[1])*(l1[0]-l2[0]) <= (l2[1]-l1[1])*(l1[0]-l3[0])

    def add(self, m, b):
        line = (m, b)
        while len(self.lines) >= 2 and self.bad(self.lines[-2], self.lines[-1], line):
            self.lines.pop()
        self.lines.append(line)

    def query_min(self, x):
        # Монотонный запрос (x возрастает): скользящий указатель O(1)
        while (self.ptr+1 < len(self.lines) and
               self.lines[self.ptr+1][0]*x + self.lines[self.ptr+1][1] <=
               self.lines[self.ptr][0]*x + self.lines[self.ptr][1]):
            self.ptr += 1
        m, b = self.lines[self.ptr]
        return m*x + b

    def query_min_any(self, x):
        # Произвольный запрос: бинарный поиск O(log n)
        lo, hi = 0, len(self.lines)-1
        while lo < hi:
            mid = (lo+hi)//2
            m1,b1 = self.lines[mid]
            m2,b2 = self.lines[mid+1]
            if m1*x+b1 <= m2*x+b2: hi = mid
            else: lo = mid+1
        m, b = self.lines[lo]
        return m*x + b
2.3 Дерево Ли Чао (Li Chao Segment Tree)

Дерево Ли Чао поддерживает операции добавления прямой и запроса минимума в точке за O(log N) каждая, без ограничений на порядок добавления и запросов. Ключевая идея: в каждом узле [l, r] хранить «победителя» в точке mid = (l+r)/2.

✓ Доказательство — Корректность дерева Ли Чао

Инвариант: в узле [l,r] хранится прямая-«победитель» в точке mid.

При добавлении прямой new в узел [l,r] с хранимой прямой cur:

Вычисляем mid = (l+r)/2. left_better = new.val(l) < cur.val(l), mid_better = new.val(mid) < cur.val(mid).

Если mid_better: меняем cur и new местами (new становится победителем).

Почему достаточно одной замены? Если new лучше cur в mid, то на [mid+1, r] cur не может быть оптимальной (две прямые пересекаются не более одного раза, new «охватывает» cur справа). Рекурсивно добавляем «проигравшую» прямую только в [l,mid] или [mid+1,r].

Каждая прямая проходит не более O(log N) узлов. ∎

Python
class LiChaoNode:
    __slots__ = ('line','left','right')
    def __init__(self): self.line=None; self.left=self.right=None

class LiChaoTree:
    def __init__(self, lo, hi):
        self.lo, self.hi = lo, hi
        self.root = LiChaoNode()

    def _add(self, node, lo, hi, line):
        if node.line is None:
            node.line = line; return
        mid = (lo + hi) // 2
        cur = node.line
        # Проверяем кто лучше в mid
        left_new  = line[0]*lo  + line[1]  < cur[0]*lo  + cur[1]
        mid_new   = line[0]*mid + line[1]  < cur[0]*mid + cur[1]
        if mid_new:
            node.line, line = line, cur  # меняем победителя
        if lo == hi: return
        if left_new != mid_new:  # 'проигравшая' прямая в левой половине
            if not node.left: node.left = LiChaoNode()
            self._add(node.left, lo, mid, line)
        else:                   # 'проигравшая' прямая в правой половине
            if not node.right: node.right = LiChaoNode()
            self._add(node.right, mid+1, hi, line)

    def add_line(self, m, b):  # прямая y = m*x + b
        self._add(self.root, self.lo, self.hi, (m, b))

    def _query(self, node, lo, hi, x):
        if node is None: return float('inf')
        mid = (lo + hi) // 2
        res = node.line[0]*x + node.line[1] if node.line else float('inf')
        if x <= mid: return min(res, self._query(node.left,  lo, mid, x))
        else:        return min(res, self._query(node.right, mid+1, hi, x))

    def query(self, x):
        return self._query(self.root, self.lo, self.hi, x)

2.4 Пример применения CHT — задача о разбиении

Задача. Разбить массив a[1..n] на не более k групп, минимизируя сумму квадратов «стоимостей» групп, где стоимость группы [l,r] = (prefix[r] - prefix[l-1])². Классический подход — ДП с CHT за O(n log n).

Python
# dp[i][j] = min стоимость разбиения a[1..i] на j групп
# Переход: dp[i][j] = min_{l<i} { dp[l][j-1] + (S[i]-S[l])^2 }
# Раскрываем: dp[l][j-1] + S[i]^2 - 2*S[i]*S[l] + S[l]^2
# = S[i]^2 + (dp[l][j-1] + S[l]^2) + (-2*S[l]) * S[i]
#             ─── b[l] ───────────────   ─ m[l] ─    ─ x ─
# => прямая с наклоном m[l] = -2*S[l] и свободным членом b[l]

def solve(a, K):
    n = len(a)
    S = [0] * (n+1)
    for i in range(n): S[i+1] = S[i] + a[i]
    INF = float('inf')
    dp = [[INF]*(n+1) for _ in range(K+1)]
    dp[0][0] = 0
    for j in range(1, K+1):
        cht = CHT_monotone()
        cht.add(-2*S[0], dp[j-1][0] + S[0]**2)
        for i in range(1, n+1):
            dp[j][i] = cht.query_min_any(S[i]) + S[i]**2
            if dp[j-1][i] < INF:
                cht.add(-2*S[i], dp[j-1][i] + S[i]**2)
    return dp[K][n]

Сумма Минковского

Выпуклые многоугольники · Применения · GJK

Сумма Минковского двух множеств A и B в ℝ² — это множество всех попарных сумм:

$$A \oplus B = \{ a + b \;|\; a \in A,\; b \in B \}$$

Для выпуклых многоугольников сумма Минковского также является выпуклым многоугольником с числом вершин не более n + m (где n, m — исходные). Это фундаментальный объект в вычислительной геометрии, теории роботов и компьютерной графике.

📐 Теорема — Структура суммы Минковского выпуклых многоугольников

Пусть A и B — выпуклые многоугольники с n и m вершинами соответственно. Тогда A⊕B — выпуклый многоугольник с не более n+m вершинами.

Ключевое свойство рёбер: рёбра A⊕B — это объединение рёбер A и B, отсортированных по полярному углу.

Доказательство: граница A⊕B параметризуется как γ(t) = α(t) + β(t), где α(t) и β(t) — точки на границах A и B с одинаковой нормалью. При обходе по нормали (вращение на 2π) каждое ребро A и B встречается ровно один раз в порядке возрастания угла нормали. Следовательно, A⊕B имеет ровно n+m рёбер (или меньше при параллельных). ∎

3.1 Алгоритм вычисления суммы Минковского — O(n+m)
Python
def minkowski_sum(P, Q):
    # P, Q — выпуклые CCW-многоугольники (вершины без дублей)
    # Шаг 1: ориентировать, начать с нижне-левой вершины
    def reorder(poly):
        i = min(range(len(poly)), key=lambda k: (poly[k][1], poly[k][0]))
        return poly[i:] + poly[:i]

    P, Q = reorder(P), reorder(Q)
    nP, nQ = len(P), len(Q)

    # Шаг 2: вычислить рёбра как векторы (dx, dy)
    def edge_vec(poly, i):
        n = len(poly)
        return (poly[(i+1)%n][0]-poly[i][0],
                poly[(i+1)%n][1]-poly[i][1])

    ep = [edge_vec(P, i) for i in range(nP)]
    eq = [edge_vec(Q, i) for i in range(nQ)]

    # Шаг 3: слияние рёбер по полярному углу (как merge в merge sort)
    result = [(P[0][0]+Q[0][0], P[0][1]+Q[0][1])]
    i = j = 0
    while i < nP or j < nQ:
        # Сравниваем ep[i] и eq[j] по углу через cross product
        if i < nP and (j >= nQ or cross((0,0), ep[i], eq[j]) > 0):
            e = ep[i]; i += 1
        elif j < nQ and (i >= nP or cross((0,0), ep[i], eq[j]) < 0):
            e = eq[j]; j += 1
        else:  # параллельные рёбра — берём оба
            e = (ep[i][0]+eq[j][0], ep[i][1]+eq[j][1])
            i += 1; j += 1
        last = result[-1]
        result.append((last[0]+e[0], last[1]+e[1]))
    result.pop()  # последняя = первая (замкнули)
    return result

def cross(O, A, B):
    return (A[0]-O[0])*(B[1]-O[1]) - (A[1]-O[1])*(B[0]-O[0])

🔧 Пример — Обнаружение столкновений (Collision Detection)

Задача: Пересекаются ли выпуклые объекты A и B?

Ключевая теорема: A ∩ B ≠ ∅ ⟺ 0 ∈ A ⊕ (−B), где −B = {−b | b ∈ B} — отражение B относительно начала координат.

Алгоритм:

1. Построить −B: отразить все вершины B.

2. Вычислить A ⊕ (−B) за O(n+m).

3. Проверить, содержит ли полученный многоугольник точку (0,0). Это делается за O(log(n+m)) бинарным поиском.

GJK-алгоритм (Gilbert-Johnson-Keerthi) работает ещё быстрее, неявно итерируя по вершинам A⊕(−B) без явного построения.

🔧 Пример — Задача о достижимости

Задача: Существуют ли a ∈ A, b ∈ B с a + b = T?

Равносильно: T ∈ A ⊕ B.

Алгоритм:

1. Построить A⊕B за O(n+m).

2. Проверить T ∈ A⊕B за O(log(n+m)).

Итого: O(n + m + log(n+m)) = O(n+m).

🔧 Пример — Сумма наборов (Three-sum / геометрическая)

Задача: Существуют ли a ∈ A, b ∈ B, c ∈ C: a+b+c = T?

1. Строим D = A ⊕ B за O(|A|+|B|).

2. Для каждого c ∈ C проверяем T−c ∈ D за O(log|D|).

Итого: O((|A|+|B|)·log(|A|+|B|) + |C|·log(|A|+|B|))

Если C тоже выпуклый набор, строим (A⊕B)⊕C за O(|A|+|B|+|C|).

def point_in_convex_polygon(poly, q):
    # poly: выпуклый CCW-многоугольник, q: точка запроса
    n = len(poly)
    if n == 0: return False
    p0 = poly[0]
    # Проверяем, в каком секторе лежит q
    if cross(p0, poly[1], q) < 0: return False
    if cross(p0, poly[n-1], q) > 0: return False
    # Бинарный поиск по треугольникам p0-poly[i]-poly[i+1]

        else: hi = mid
    # Проверяем треугольник p0-poly[lo]-poly[lo+1]

Метод сканирующей прямой (Scanline)

События · Активные отрезки · Пересечения · Площадь объединения

Метод сканирующей прямой (scanline) — фундаментальная техника вычислительной геометрии. Идея: вертикальная прямая движется слева направо по плоскости. В каждый момент нас интересует множество объектов, пересекаемых прямой в данной точке x — активное множество. При движении прямой состав активного множества меняется только в точках событий (начало/конец отрезка, пересечение).

📐 Определение — Сканирующая прямая

Пусть на плоскости дано множество геометрических объектов (отрезков, прямоугольников, окружностей). Сканирующая прямая — вертикальная прямая x = const, движущаяся слева направо.

Точка события — координата x, где изменяется активное множество:

• Начало отрезка/прямоугольника (добавление в активное множество)

• Конец отрезка/прямоугольника (удаление из активного множества)

• Пересечение двух отрезков (изменение порядка в активном множестве)

Сложность: O((n + k) log n), где k — число событий (пересечений).

4.1 Общая схема алгоритма
📐 Теорема — Корректность метода сканирующей прямой

Рассмотрим произвольное свойство P геометрической конфигурации, зависящее только от пересечения сканирующей прямой x = x₀ с объектами.

Утверждение: если P изменяется только в точках событий, то достаточно проверить P только в этих точках для полного анализа.

Доказательство: между двумя последовательными точками событий xᵢ и xᵢ₊₁ состав активного множества не меняется. Если объекты — отрезки, то порядок их пересечения с прямой x = const сохраняется (иначе было бы пересечение — точка события). Следовательно, любое свойство, зависящее только от активного множества и порядка, постоянно на (xᵢ, xᵢ₊₁). ∎

4.2 Пересечение множества отрезков — O((n+k) log n)

Задача. Найти все точки пересечения n отрезков на плоскости (общего положения: нет вертикальных, нет троек через одну точку).

Идея. Сканирующая прямая поддерживает активные отрезки, упорядоченные по y-координате пересечения с прямой. Пересечение возможно только между соседними в этом порядке отрезками. При добавлении нового отрезка проверяем пересечения только с соседями.

✓ Доказательство — Почему проверять только соседей

Лемма: если два отрезка s₁ и s₂ пересекаются в точке p = (xₚ, yₚ), то непосредственно перед xₚ они были соседями в упорядочении активного множества по y.

Доказательство: пусть s₁ и s₂ пересекаются в p. При x = xₚ - ε (немного левее пересечения) отрезки не пересечены. Пусть s₁ ниже s₂ в этой точке. Если между ними есть отрезок s₃, то s₃ пересекает либо s₁, либо s₂ левее xₚ (т.к. все три проходят через область между s₁ и s₂). Противоречие с общим положением (пересечение левее уже обнаружено). Следовательно, s₁ и s₂ — соседи. ∎

Следствие: достаточно проверять пересечения только при изменении соседства: добавление отрезка, удаление отрезка, пересечение двух соседей (меняем порядок).

from sortedcontainers import SortedList  # или treap / RB-tree

class Segment:
    def __init__(self, x1, y1, x2, y2):
        self.x1, self.y1 = x1, y1
        self.x2, self.y2 = x2, y2
        # y = k*x + b
        self.k = (y2-y1)/(x2-x1) if x2 != x1 else 0
        self.b = y1 - self.k*x1

    def y_at(self, x):
        return self.k*x + self.b

    def intersect(self, other):
        # Решаем систему: k1*x+b1 = k2*x+b2
        if self.k == other.k: return None
        x = (other.b - self.b) / (self.k - other.k)
        if max(self.x1, other.x1) <= x <= min(self.x2, other.x2):
            return (x, self.y_at(x))
        return None

def segment_intersections(segments):
    """Найти все пересечения n отрезков. O((n+k) log n)."""
    events = []
    for i, s in enumerate(segments):
        if s.x1 > s.x2:  # ориентируем слева направо
            s.x1, s.x2 = s.x2, s.x1
            s.y1, s.y2 = s.y2, s.y1
            s.k = -s.k; s.b = s.y1 - s.k*s.x1
        events.append((s.x1, 0, i))   # 0 = начало
        events.append((s.x2, 1, i))   # 1 = конец
    events.sort()

    # Активное множество: отрезки, упорядоченные по y_at(current_x)
    # Ключ: (y_at(x), id) для уникальности
    active = SortedList(key=lambda i: segments[i].y_at(current_x))
    current_x = 0
    result = []

    for x, typ, idx in events:
        current_x = x
        s = segments[idx]
        if typ == 0:  # добавление
            pos = active.bisect(idx)
            # Проверяем пересечения с соседями
            if pos > 0:
                p = s.intersect(segments[active[pos-1]])
                if p: result.append(p)
            if pos < len(active):
                p = s.intersect(segments[active[pos]])
                if p: result.append(p)
            active.add(idx)
        else:  # удаление
            pos = active.index(idx)
            # После удаления бывшие соседи становятся соседями
            if pos > 0 and pos+1 < len(active):
                p = segments[active[pos-1]].intersect(
                    segments[active[pos+1]])
                if p: result.append(p)
            active.remove(idx)
    return result
4.3 Площадь объединения прямоугольников — O(n log n)

Задача. Даны оси-выровненные прямоугольники [x₁,x₂] × [y₁,y₂]. Найти площадь их объединения.

Идея. Сканирующая прямая движется по x. На каждом отрезке [xᵢ, xᵢ₊₁] между событиями активное множество — набор y-интервалов. Площадь = сумма (xᵢ₊₁ − xᵢ) × (длина объединения y-интервалов).

✓ Доказательство — Корректность площади через scanline

Разобьём объединение прямоугольников на вертикальные полосы [xᵢ, xᵢ₊₁], где xᵢ — отсортированные уникальные x-координаты границ.

На полосе [xᵢ, xᵢ₊₁] множество активных y-интервалов постоянно. Пусть Lᵢ — длина объединения этих интервалов. Тогда вклад полосы в площадь: (xᵢ₊₁ − xᵢ) × Lᵢ.

Сумма по всем полосам даёт точную площадь объединения, т.к. полосы не пересекаются и покрывают всё объединение. ∎

Python
def union_area_rectangles(rects):
    """Площадь объединения оси-выровненных прямоугольников."""
    # rects: [(x1, y1, x2, y2), ...]
    if not rects: return 0

    # События: (x, тип, y1, y2), тип: +1 = начало, -1 = конец
    events = []
    for x1, y1, x2, y2 in rects:
        events.append((x1, 1, y1, y2))
        events.append((x2, -1, y1, y2))
    events.sort()

    # Дерево отрезков или line sweep по y
    # Упрощённая версия: считаем длину покрытия активных интервалов
    from collections import defaultdict
    active = defaultdict(int)  # y-координата -> счётчик покрытия
    prev_x = events[0][0]
    area = 0

    def covered_length():
        # Длина объединения активных y-интервалов
        if not active: return 0
        pts = sorted(active.keys())
        total = 0
        cur_cover = 0
        prev_y = pts[0]
        for y in pts:
            if cur_cover > 0:
                total += y - prev_y
            cur_cover += active[y]
            prev_y = y
        return total

    for x, typ, y1, y2 in events:
        dx = x - prev_x
        if dx > 0:
            area += dx * covered_length()
            prev_x = x
        active[y1] += typ
        active[y2] -= typ
        if active[y1] == 0: del active[y1]
        if active[y2] == 0: del active[y2]

    return area

Классическая задача о силуэте зданий — это scanline, где активное множество содержит высоты, а нас интересует максимум.

import heapq

def scanline_skyline(buildings):
    """Силуэт зданий через scanline + max-heap. O(n log n)."""
    # События: (x, тип, высота, конец)
    events = []
    for l, r, h in buildings:
        events.append((l, -h, r))   # -h: начало (h > 0 => приоритет)
        events.append((r,  0, 0))   #  0: конец
    events.sort()

    result = []
    # Max-heap через min-heap с отрицательными значениями
    heap = [(0, float('inf'))]  # sentinel: высота 0 до бесконечности
    prev_max = 0

    for x, neg_h, end in events:
        if neg_h < 0:  # начало здания
            heapq.heappush(heap, (neg_h, end))
        # Ленивое удаление: убираем здания, уже закончившиеся до x
        while heap[0][1] <= x:
            heapq.heappop(heap)
        # Текущий максимум высот
        cur_max = -heap[0][0]
        if cur_max != prev_max:
            result.append((x, cur_max))
            prev_max = cur_max
    return result
✓ Доказательство — Корректность lazy deletion в skyline

Утверждение: в момент запроса heap[0] — активное здание с максимальной высотой.

Доказательство: при запросе в точке x удаляем элементы (neg_h, end) с end ≤ x. Эти здания закончились до x, поэтому не влияют на высоту в x. ✓

Оставшиеся элементы — здания с end > x, т.е. активные в x. heap[0] — минимальный neg_h среди них, т.е. максимальный h. ✓

Каждое здание добавляется и удаляется из heap ровно 1 раз. Суммарная сложность: O(n log n). ∎

# Сжатие координат + дерево отрезков для scanline
def scanline_segtree(rects):
    """Площадь объединения через scanline + segment tree."""
    # Собрать все уникальные x-координаты
    xs = sorted(set(x for x1,y1,x2,y2 in rects for x in (x1, x2)))
    idx = {x: i for i, x in enumerate(xs)}
    m = len(xs)
    tree = [0] * (4 * m)  # дерево отрезков max

    def update(node, lo, hi, l, r, h):
        if r <= lo or hi <= l: return
        if l <= lo and hi <= r:
            tree[node] = max(tree[node], h); return
        mid = (lo+hi)//2
        update(2*node, lo, mid, l, r, h)
        update(2*node+1, mid, hi, l, r, h)

    def build(node, lo, hi):
        if hi - lo == 1: return
        mid = (lo+hi)//2
        build(2*node, lo, mid)
        build(2*node+1, mid, hi)
        tree[node] = max(tree[2*node], tree[2*node+1])

    for x1, y1, x2, y2 in rects:
        update(1, 0, m, idx[x1], idx[x2], y2-y1)
    build(1, 0, m)  # propagate
    # Собрать результат
    result = []
    prev = 0
    for i, x in enumerate(xs):
        # query point i
        h = 0  # упрощённо; полная реализация требует query(i,i+1)
        if h != prev: result.append((x, h)); prev = h
    return result

DFT — Дискретное Преобразование Фурье

Определение · Матричная форма · Свойства · Обращение

Дискретное преобразование Фурье (DFT) переводит последовательность n комплексных чисел в n коэффициентов по примитивным корням из единицы. Это дискретный аналог преобразования Фурье для непрерывных функций.

📐 Определение — DFT и IDFT

Пусть a = (a₀, a₁, ..., a_{n-1}), ω_n = e^{2πi/n} (примитивный корень).

DFT:

$$A[k] = \sum_{j=0}^{n-1} a[j] \cdot \omega_n^{jk}, \quad k = 0, 1, \dots, n-1$$

IDFT:

$$a[j] = \frac{1}{n} \sum_{k=0}^{n-1} A[k] \cdot \omega_n^{-jk}, \quad j = 0, \dots, n-1$$

Матричная форма: A = F_n · a, где F_n[j,k] = ω_n^{jk}. F_n — матрица ДПФ; F_n^{-1} = (1/n) · conj(F_n) (F_n — унитарная с норм. 1/√n).

5.1 Доказательство формулы обращения IDFT
✓ Доказательство — IDFT корректно обращает DFT

Покажем: (IDFT ∘ DFT)(a) = a.

$$(\text{IDFT}\cdot\text{DFT}(a))[j] = \frac{1}{n} \sum_{k=0}^{n-1} A[k] \cdot \omega_n^{-jk}$$

$$= \frac{1}{n} \sum_k \left[\sum_m a[m] \omega_n^{mk}\right] \omega_n^{-jk} = \frac{1}{n} \sum_m a[m] \sum_k \omega_n^{k(m-j)}$$

Вычислим внутреннюю сумму S(d) = Σ_{k=0}^{n-1} ω_n^{k·d}:

Если d ≡ 0 (mod n): S(d) = Σ 1^k = n

Если d ≢ 0 (mod n): S(d) = (ω_n^{dn} - 1)/(ω_n^d - 1) = 0 (геом. прогрессия со знаменателем ≠ 1, сумма = 0)

Следовательно:

$$(\text{IDFT}\cdot\text{DFT}(a))[j] = \frac{1}{n} \cdot a[j] \cdot n = a[j] \;\blacksquare$$

5.2 Ключевые свойства DFT
📐 Теорема — Теорема о свёртке (Convolution Theorem)

Линейная свёртка: (a*b)[k] = Σ_{j=0}^{k} a[j]·b[k-j]

Утверждение: DFT(a * b)[k] = DFT(a)[k] · DFT(b)[k]

Доказательство:

$$\text{DFT}(a*b)[k] = \sum_n \left(\sum_j a[j]\cdot b[n-j]\right) \omega^{nk}$$

$$= \sum_j a[j] \sum_n b[n-j] \omega^{nk} = \sum_j a[j] \omega^{jk} \sum_m b[m] \omega^{mk}$$

(замена m = n-j, ω^{nk} = ω^{jk}·ω^{mk})

$$= \text{DFT}(a)[k] \cdot \text{DFT}(b)[k] \;\blacksquare$$

Следствие: умножение полиномов через DFT:

1. Вычислить A = DFT(a), B = DFT(b) — O(n log n)

2. C[k] = A[k]·B[k] для всех k — O(n)

3. c = IDFT(C) — O(n log n)

📐 Теорема — Теорема Парсеваля (Parseval's identity)

$$\sum_{j=0}^{n-1} |a[j]|^2 = \frac{1}{n} \sum_{k=0}^{n-1} |A[k]|^2$$

Смысл: DFT сохраняет L²-норму (с точностью до 1/√n).

Доказательство:

$$\sum_j |a[j]|^2 = \langle a, a \rangle = \langle F^{-1}A, F^{-1}A \rangle$$

$$= \langle A, (F\cdot F^H/n)\cdot A \rangle = \frac{1}{n}\langle A,A \rangle = \frac{1}{n}\sum_k|A[k]|^2 \;\blacksquare$$

import cmath, math

def dft(a, invert=False):
    """Наивный DFT за O(n²). Для демонстрации и проверки."""
    n = len(a)
    A = []
    for k in range(n):
        Ak = 0
        for j in range(n):
            angle = 2 * math.pi * j * k / n
            if invert: angle = -angle
            Ak += a[j] * cmath.exp(1j * angle)
        if invert: Ak /= n
        A.append(Ak)
    return A

FFT — Быстрое Преобразование Фурье

Кули-Тьюки · Бабочка · Bit-reversal · Оптимизации

FFT (Fast Fourier Transform) вычисляет DFT за O(n log n) используя принцип «разделяй-и-властвуй». Алгоритм Кули–Тьюки (1965) — наиболее распространённая реализация для n = 2^k.

6.1 Вывод алгоритма Кули–Тьюки
✓ Доказательство — Формула «бабочки» (butterfly)

Дано: DFT размера n, n = 2^k. Разбиваем a на чётные/нечётные:

a_even = [a[0], a[2], a[4], ...] (n/2 элементов)

a_odd = [a[1], a[3], a[5], ...] (n/2 элементов)

$$A[k] = \sum_{j=0}^{n-1} a[j] \cdot \omega_n^{jk}$$

$$= \sum_m a[2m] \cdot \omega_n^{2mk} + \sum_m a[2m+1] \cdot \omega_n^{(2m+1)k}$$

$$= \sum_m a_{\text{even}}[m] \cdot (\omega_n^2)^{mk} + \omega_n^k \cdot \sum_m a_{\text{odd}}[m] \cdot (\omega_n^2)^{mk}$$

Замечаем: ω_n² = e^{2πi·2/n} = e^{2πi/(n/2)} = ω_{n/2}

$$\Rightarrow A[k] = E[k] + \omega_n^k \cdot O[k], \quad k = 0..n/2-1$$

$$\Rightarrow A[k+n/2] = E[k] - \omega_n^k \cdot O[k], \quad k = 0..n/2-1$$

где E = DFT_{n/2}(a_even), O = DFT_{n/2}(a_odd)

Операция (E[k], O[k]) → (E[k]+w·O[k], E[k]-w·O[k]) — «бабочка». ∎

6.2 Итеративный FFT с bit-reversal permutation

Рекурсивный FFT имеет накладные расходы на вызовы. Итеративная версия сначала переставляет элементы в «bit-reversed» порядок, затем применяет бабочки снизу вверх по слоям.

📐 Теорема — Корректность bit-reversal permutation

При рекурсивном разделении чётных/нечётных n/2 раза, элемент с индексом j оказывается на позиции rev(j), где rev(j) — j с битами, записанными в обратном порядке.

Пример (n=8, 3 бита):

j=0: 000 → 000: позиция 0

j=1: 001 → 100: позиция 4

j=2: 010 → 010: позиция 2

j=3: 011 → 110: позиция 6

j=4: 100 → 001: позиция 1

Корректность: на каждом уровне рекурсии чётные/нечётные = 0/1 бит в младшем разряде. После log₂(n) уровней биты оказываются обращены. ∎

Python
import cmath, math

def fft(a, invert=False):
    n = len(a)
    assert n & (n-1) == 0, 'n must be power of 2'
    # Шаг 1: bit-reversal permutation
    j = 0
    for i in range(1, n):
        bit = n >> 1
        while j & bit:
            j ^= bit
            bit >>= 1
        j ^= bit
        if i < j:
            a[i], a[j] = a[j], a[i]
    # Шаг 2: послойные бабочки
    length = 2
    while length <= n:
        # Корень из единицы для текущего размера
        ang = 2 * math.pi / length * (-1 if invert else 1)
        w_start = cmath.exp(1j * ang)
        for i in range(0, n, length):
            w = 1 + 0j
            half = length // 2
            for k in range(half):
                u = a[i + k]
                v = a[i + k + half] * w
                a[i + k]        = u + v  # верхняя 'нога' бабочки
                a[i + k + half] = u - v  # нижняя 'нога' бабочки
                w *= w_start
        length <<= 1
    if invert:
        inv_n = 1 / n
        for i in range(n): a[i] *= inv_n

def poly_multiply(p, q):
    """Умножение полиномов через FFT за O(n log n)."""
    result_len = len(p) + len(q) - 1
    n = 1
    while n < result_len: n <<= 1  # следующая степень 2
    fa = [complex(x) for x in p] + [0j]*(n - len(p))
    fb = [complex(x) for x in q] + [0j]*(n - len(q))
    fft(fa); fft(fb)
    fc = [a*b for a, b in zip(fa, fb)]
    fft(fc, invert=True)
    return [round(c.real) for c in fc[:result_len]]

✓ Доказательство — FFT: T(n) = O(n log n)

Рекуррентность: T(n) = 2·T(n/2) + O(n)

— 2 рекурсивных вызова размера n/2

— O(n) для комбинирования (n/2 бабочек)

Master Theorem (случай 2): a=2, b=2, f(n)=n:

$$n^{\log_b a} = n^1 = \Theta(f(n)) \quad\Rightarrow\quad T(n) = \Theta(n \log n)$$

Итеративная версия: log₂(n) слоёв, в каждом n/2 бабочек = n/2 умножений. Итого: (n/2)·log₂(n) умножений и n·log₂(n) сложений. O(n log n) арифметических операций. ∎

Нижняя оценка: O(n log n) для умножения полиномов оптимальна (под моделью с алгебраическими деревьями вычислений).

6.4 Практические оптимизации FFT
💡 Замечание — Вещественный FFT (real FFT)

Если входные данные вещественные (a[j] ∈ ℝ), можно сэкономить в ~2 раза:

Объединяем два вещественных вектора p и q в один комплексный: z[j] = p[j] + i·q[j]

NTT — Числовое Преобразование Фурье

NTT-friendly primes · Примитивный корень · CRT

NTT (Number Theoretic Transform) — аналог FFT, работающий в кольце вычетов Z_p. Результаты — точные целые числа без ошибок округления. NTT применяется везде, где коэффициенты полиномов — целые числа по некоторому модулю.

📐 Определение — Условия применимости NTT

Требования:

1. p — простое число

2. n | (p-1) — n должна делить p-1 (чтобы существовал корень порядка n)

3. g — примитивный корень по модулю p

Тогда аналог ω_n = g^{(p-1)/n} mod p:

ω_n^n = g^{p-1} ≡ 1 (mod p) — по малой теореме Ферма

Порядок ω_n равен в точности n — т.к. g примитивный корень

NTT формулы идентичны FFT, но все вычисления mod p:

$$A[k] = \sum_{j=0}^{n-1} a[j] \cdot \omega_n^{jk} \pmod p$$

$$a[j] = n^{-1} \cdot \sum_{k=0}^{n-1} A[k] \cdot \omega_n^{-jk} \pmod p$$

n^{-1} = n^{p-2} mod p — обратный элемент по Ферма

7.1 Теоретическое обоснование NTT
✓ Доказательство — NTT корректно вычисляет DFT в Z_p

Лемма о геометрической сумме в Z_p:

$$\sum_{k=0}^{n-1} \omega_n^{k\cdot d} \equiv n\cdot[d\equiv 0\pmod n] \pmod p$$

Доказательство: если d≡0(mod n): все слагаемые = 1, сумма = n.

Если d≢0(mod n): ω_n^d ≠ 1 (порядок ω_n = n, d не кратно n).

$$S = \sum_{k=0}^{n-1} r^k = \frac{r^n - 1}{r - 1} \equiv 0 \pmod p$$

т.к. r^n = ω_n^{dn} = (ω_n^n)^d = 1^d = 1, и r-1 ≠ 0 (mod p). ✓

Корректность INTT:

$$(\text{INTT}\cdot\text{NTT}(a))[j] = n^{-1} \sum_k \left[\sum_m a[m]\omega^{km}\right] \omega^{-jk}$$

$$= n^{-1} \sum_m a[m] \sum_k \omega^{k(m-j)} = n^{-1}\cdot a[j]\cdot n = a[j] \;\blacksquare$$

MOD = 998_244_353
PRIM_ROOT = 3

def power(base, exp, mod=MOD):
    """Быстрое возведение в степень — O(log exp)."""
    r = 1; base %= mod
    while exp > 0:
        if exp & 1: r = r * base % mod
        base = base * base % mod
        exp >>= 1
    return r

def ntt(a, invert=False, mod=MOD):
    n = len(a)
    assert n & (n-1) == 0
    # bit-reversal (идентично FFT)
    j = 0
    for i in range(1, n):
        bit = n >> 1
        while j & bit: j ^= bit; bit >>= 1
        j ^= bit
        if i < j: a[i], a[j] = a[j], a[i]
    # послойные бабочки в Z_mod
    length = 2
    while length <= n:
        # Примитивный корень порядка length
        w = power(PRIM_ROOT, (mod-1)//length, mod)
        if invert: w = power(w, mod-2, mod)  # обратный элемент
        for i in range(0, n, length):
            wn = 1
            for k in range(length >> 1):
                u = a[i+k]
                v = a[i+k + length//2] * wn % mod
                a[i+k]              = (u + v) % mod
                a[i+k + length//2]  = (u - v) % mod
                wn = wn * w % mod
        length <<= 1
    if invert:
        inv_n = power(n, mod-2, mod)
        for i in range(n): a[i] = a[i] * inv_n % mod

def poly_multiply_ntt(p, q, mod=MOD):
    n = 1
    while n < len(p)+len(q): n <<= 1
    a = p[:] + [0]*(n-len(p))
    b = q[:] + [0]*(n-len(q))
    ntt(a, mod=mod); ntt(b, mod=mod)
    c = [x*y % mod for x,y in zip(a, b)]
    ntt(c, invert=True, mod=mod)
    return c[:len(p)+len(q)-1]

Если результирующие коэффициенты могут превышать MOD (например, умножаем полиномы с большими коэффициентами), используют несколько NTT-простых и Китайскую теорему об остатках (CRT). Стандартная тройка: p₁ = 998244353, p₂ = 985661441, p₃ = 754974721.

Python
def crt2(r1, m1, r2, m2):
    """CRT для двух модулей: найти x ≡ r1 (mod m1), x ≡ r2 (mod m2)."""
    # x = r1 + m1 * t,  t = (r2-r1) * m1^{-1} (mod m2)
    t = (r2 - r1) % m2 * power(m1, m2-2, m2) % m2
    return r1 + m1 * t  # результат в [0, m1*m2)

def poly_multiply_arbitrary_mod(p, q, MOD_OUT):
    """Умножение полиномов mod MOD_OUT (любой, не обязательно NTT-friendly)."""
    P1, P2 = 998_244_353, 985_661_441
    # NTT по двум простым
    c1 = poly_multiply_ntt(p, q, P1)
    c2 = poly_multiply_ntt(p, q, P2)
    # CRT: c[i] mod (P1*P2), затем mod MOD_OUT
    result = []
    for a1, a2 in zip(c1, c2):
        val = crt2(a1, P1, a2, P2) % MOD_OUT
        result.append(val)
    return result

AND / OR / XOR Свёртки

SOS DP · Walsh-Hadamard · Subset Sum Convolution

Помимо линейной свёртки полиномов, существуют подмножественные свёртки над операциями AND, OR, XOR на битовых масках. Они возникают в задачах о подмножествах, покрытиях, теории кодирования и вероятностных алгоритмах.

$$C[k] = \sum_{i \oplus j = k} A[i] \cdot B[j] \quad\text{(XOR-свёртка)}$$
$$C[k] = \sum_{i \& j = k} A[i] \cdot B[j] \quad\text{(AND-свёртка)}$$
$$C[k] = \sum_{i \mid j = k} A[i] \cdot B[j] \quad\text{(OR-свёртка)}$$
8.1 OR-свёртка и преобразование по подмножествам (SOS DP)

Зета-преобразование по OR (Sum over Subsets): Â[mask] = Σ_{sub ⊆ mask} A[sub] — сумма A по всем подмножествам mask. Это «обновление снизу вверх» по решётке подмножеств (Boolean lattice).

✓ Доказательство — Корректность зета-преобразования по OR

После обработки бита i: Â[mask] = Σ_{sub: биты > i равны bits>i(mask), биты ≤ i: sub ⊆ mask} A[sub]

Инвариант: перед обработкой бита i:

Â[mask] = Σ_{sub: sub[bits ≥ i] = mask[bits ≥ i], sub[bits<i]⊆mask[bits<i]} A[sub]

После n итераций (по битам 0..n-1):

Â[mask] = Σ_{sub ⊆ mask} A[sub] — суммирование по всем подмножествам.

Переход (обработка бита i):

Если mask & (1<<i): Â[mask] += Â[mask ^ (1<<i)]

(добавляем вклад масок, у которых бит i = 0 при тех же остальных битах). ∎

Python
def zeta_or(f):
    """Зета-преобразование: f[mask] = Σ_{sub ⊆ mask} f[sub]"""
    f = f[:]
    n = len(f)
    i = 1
    while i < n:
        for mask in range(n):
            if mask & i:  # бит i установлен
                f[mask] += f[mask ^ i]  # добавляем без бита i
        i <<= 1
    return f

def mobius_or(f):
    """Обратное (преобразование Мёбиуса): f[mask] = Σ_{sub ⊆ mask} (-1)^|mask\sub| f[sub]"""
    f = f[:]
    n = len(f)
    i = 1
    while i < n:
        for mask in range(n):
            if mask & i:
                f[mask] -= f[mask ^ i]
        i <<= 1
    return f

def or_convolution(A, B):
    hatA, hatB = zeta_or(A), zeta_or(B)
    hatC = [a*b for a,b in zip(hatA, hatB)]
    return mobius_or(hatC)

8.3 XOR-свёртка и преобразование Уолша–Адамара (WHT)

WHT (Walsh-Hadamard Transform) — аналог FFT для группы (Z/2Z)^n по операции XOR. Матрица преобразования — матрица Адамара: строится рекурсивно как тензорное произведение.

✓ Доказательство — WHT корректно вычисляет XOR-свёртку

Символы группы (Z/2Z)^n: χ_k(j) = (-1)^{popcount(j AND k)}

$$\hat A[k] = \sum_j A[j] \cdot \chi_k(j) = \sum_j A[j] \cdot (-1)^{\text{popcount}(j \& k)}$$

Ключевое свойство мультипликативности:

$$\chi_k(j \oplus m) = (-1)^{\text{popcount}((j \oplus m) \& k)} = (-1)^{\text{popcount}(j\&k) + \text{popcount}(m\&k)} = \chi_k(j) \cdot \chi_k(m)$$

Тогда для C[i] = Σ_{j XOR m = i} A[j]·B[m]:

$$\hat C[k] = \sum_i C[i]\cdot\chi_k(i) = \sum_{j,m} A[j]\cdot B[m]\cdot\chi_k(j \oplus m)$$

$$= \sum_j A[j]\chi_k(j) \cdot \sum_m B[m]\chi_k(m) = \hat A[k] \cdot \hat B[k] \;\blacksquare$$

Обращение: WHT^{-1} = (1/n)·WHT (WHT самообратно с нормировкой). ∎

Python
def fwht(f, invert=False):
    """Fast Walsh-Hadamard Transform. f изменяется in-place."""
    f = f[:]
    n = len(f)
    h = 1
    while h < n:
        for i in range(0, n, h*2):
            for j in range(i, i+h):
                x, y = f[j], f[j+h]
                f[j]   = x + y   # верхняя 'нога' бабочки Адамара
                f[j+h] = x - y   # нижняя 'нога'
        h <<= 1
    if invert:
        for i in range(n): f[i] //= n  # или /= n для нецелых
    return f

def xor_convolution(A, B):
    hatA = fwht(A)
    hatB = fwht(B)
    hatC = [a*b for a,b in zip(hatA, hatB)]
    return fwht(hatC, invert=True)

8.4 Подмножественная свёртка (Subset Sum Convolution)

Subset Sum Convolution (SSC) вычисляет строгую свёртку по непересекающимся подмножествам: C[S] = Σ_{T ⊆ S} A[T]·B[S\\T]. Ключевая идея — добавить «ранговое» измерение popcount и применить зета-преобразование по каждому рангу отдельно.

✓ Доказательство — Корректность Subset Sum Convolution

Идея: C[S] = Σ_{T ⊆ S} A[T]·B[S\\T] = Σ_{T ⊆ S, T∩(S\\T)=∅} A[T]·B[S\\T]

Вводим ранговые версии: f̂[r][mask] = A[mask] если |mask|=r, иначе 0. Аналогично ĝ[r][mask] = B[mask] если |mask|=r, иначе 0.

После зета-преобразования по OR:

F[r][mask] = Σ_{sub⊆mask, |sub|=r} A[sub]

G[r][mask] = Σ_{sub⊆mask, |sub|=r} B[sub]

Свёртка по рангам: H[r][mask] = Σ_{j=0}^{r} F[j][mask]·G[r-j][mask]

После обратного (Мёбиус) преобразования:

h[r][mask] = Σ_{sub⊆mask, |sub|=r} Σ_{j} A[sub_j]·B[sub_{r-j}]

C[S] = H[|S|][S], поскольку |T|+|S\\T| = |S| и T∩(S\\T) = ∅. ∎

def subset_sum_convolution(A, B):
    """C[S] = Σ_{T ⊆ S} A[T]*B[S \ T]"""
    from math import log2
    N = len(A)
    K = int(log2(N))  # N = 2^K

    # Ранговые версии: fA[r][mask], fB[r][mask]
    fA = [[0]*N for _ in range(K+1)]
    fB = [[0]*N for _ in range(K+1)]
    for mask in range(N):
        pc = bin(mask).count('1')
        fA[pc][mask] = A[mask]
        fB[pc][mask] = B[mask]

    # Зета-преобразование по OR для каждого ранга
    for r in range(K+1):
        fA[r] = zeta_or(fA[r])
        fB[r] = zeta_or(fB[r])

    # Свёртка по рангам (для каждой маски — полиномиальное умножение)
    fC = [[0]*N for _ in range(K+1)]
    for mask in range(N):
        for j in range(K+1):
            for r in range(j, K+1):
                fC[r][mask] += fA[j][mask] * fB[r-j][mask]

    # Обратное преобразование (Мёбиус по OR)
    for r in range(K+1):
        fC[r] = mobius_or(fC[r])

    # Ответ: C[S] = fC[popcount(S)][S]
    return [fC[bin(mask).count('1')][mask] for mask in range(N)]

💡 Замечание — Выбор свёртки для задачи

• Умножение полиномов (целые коэффициенты, mod M) → NTT

• Умножение полиномов (вещественные, приблизительно) → FFT

• Задачи о подмножествах (существование пересечения) → AND

• Задачи о покрытии (покрыть множество) → OR

• Задачи о чётности / XOR-пути → XOR (WHT)

• Ровно T подмножеств / непересекающиеся покрытия → Subset Sum

Hint: OR-свёртка + мёбиус = SOS DP, одна из самых частых техник на олимпиадных задачах с битовыми масками.
