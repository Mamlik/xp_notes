Теория игр: графы, сумма игр, Ним и теоремы Шпрага—Гранди

Конспект для контрольной по алгоритмам: равноправные конечные игры с полной информацией, оптимальная стратегия, терминальные состояния, сумма игр, эквивалентность по Гранди, функция Grundy, MEX и XOR-стратегия для Нима.

1. Представление игры в виде графа

В этой лекции рассматриваются не любые игры, а специальный класс игр, который особенно важен в алгоритмах. Это конечные равноправные игры с полной информацией без случайности.

Определение

Игру можно представить как ориентированный граф состояний. Вершина графа — это состояние игры. Ребро A → B означает, что из состояния A одним ходом можно перейти в состояние B.

A → B → T

Если игра находится в вершине A, то игрок, чей сейчас ход, выбирает одно исходящее ребро и переводит игру в одну из достижимых вершин.

1.1. Какие игры рассматриваем

Свойство Что означает
Равноправные У обоих игроков одинаковые доступные ходы из одного и того же состояния. Нет ходов, доступных только первому или только второму игроку.
Конечные Игра обязательно заканчивается за конечное число ходов.
Ациклический граф Нельзя ходами вернуться в уже посещённое состояние. Поэтому нет бесконечных партий.
Ходы по очереди Игроки делают ровно по одному ходу, затем право хода переходит сопернику.
Полная информация Оба игрока видят всё состояние игры. Нет скрытых карт, случайных бросков и неизвестной информации.
Оптимальность зависит только от состояния Дальнейшая стратегия определяется текущей вершиной графа, а не историей того, как мы в неё попали.
Выигрыш зависит только от состояния Если состояние терминальное, оно само определяет, что текущий игрок проиграл.
Важно

Это именно impartial combinatorial games: игры без случайности, без ничьих, с одинаковыми возможностями для игроков. Шахматы сюда напрямую не попадают: там разные фигуры у белых и чёрных, а также есть ничьи и сложные правила окончания.

2. Терминальные состояния, выигрыш и проигрыш

Терминальное состояние

Терминальное состояние — состояние, из которого нет доступных ходов. По нормальному правилу игры игрок, который не может сделать ход, проигрывает.

Поэтому терминальная вершина всегда является проигрышной для игрока, который должен ходить.

2.1. Выигрышные и проигрышные состояния

P- и N-позиции

Часто используют обозначения:

  • P-position — previous player winning, то есть позиция проигрышна для текущего игрока;
  • N-position — next player winning, то есть позиция выигрышна для текущего игрока.

В лекции значение игры обозначается:

v(A) ∈ { -1, 0, 1 }

В рамках обычных конечных игр без ничьих реально важны два значения:

Значение Смысл
v(A) = -1 Проигрышная позиция для текущего игрока.
v(A) = 1 Выигрышная позиция для текущего игрока.
v(A) = 0 Обычно соответствует ничейным или более общим ситуациям. В рассматриваемом ациклическом классе без ничьих это значение фактически не нужно.

2.2. Рекурсивное правило

Критерий выигрышности

Состояние проигрышно тогда и только тогда, когда все ходы из него ведут в выигрышные состояния. Состояние выигрышно тогда и только тогда, когда существует хотя бы один ход в проигрышное состояние.

Доказательство

Если из состояния A есть ход в проигрышное состояние B, то текущий игрок делает этот ход. После этого соперник оказывается в проигрышной позиции, значит при оптимальной игре проиграет. Поэтому A выигрышна.

Если же все ходы из A ведут в выигрышные состояния, то текущий игрок обязан перевести игру в состояние, из которого соперник имеет выигрышную стратегию. Значит, текущий игрок проигрывает. Поэтому A проигрышна.

Алгоритмическая идея

Для конечного ациклического графа значения состояний можно считать динамически в обратном топологическом порядке: сначала терминальные вершины, затем вершины, из которых все переходы уже посчитаны.

3. Сумма игр

Одна из главных идей лекции — рассматривать не только одну игру, но и несколько независимых игр, в одну из которых игрок делает ход на каждом своём ходу.

Сумма игр

Сумма игр A + B — это игра, состояние которой состоит из пары состояний (A, B). За один ход игрок выбирает ровно одну компоненту и делает ход только в ней.

То есть из A + B можно перейти либо в A' + B, если A → A', либо в A + B', если B → B'.

Пример

Пусть есть две кучки Нима: 3 камня и 5 камней. Игра *3 + *5 означает, что за ход можно уменьшить либо первую кучку, либо вторую, но нельзя за один ход менять обе кучки сразу.

Терминальное состояние суммы достигается только тогда, когда терминальны все компоненты. Например, *0 + *0 — проигрышная игра без ходов.

4. Базовые свойства суммы игр

Сумма игр ведёт себя очень похоже на обычное сложение, но нужно понимать это не как равенство графов буквально, а как естественную изоморфность игр.

4.1. Коммутативность

Свойство

A + B эквивалентна B + A.

Доказательство

Состояние A + B — это пара (a, b). Состояние B + A — это пара (b, a). Между ними есть очевидное взаимно однозначное соответствие: поменять компоненты местами.

Если в первой игре можно сделать ход в компоненте A, то во второй игре это соответствует ходу во второй компоненте. Если можно сделать ход в B, то это соответствует ходу в первой компоненте. Значит, структура доступных ходов совпадает.

4.2. Ассоциативность

Свойство

(A + B) + C эквивалентна A + (B + C).

Доказательство

Обе игры фактически описывают тройку независимых компонент (A, B, C). За ход игрок выбирает одну из трёх компонент и делает ход в ней. Скобки лишь меняют формальную запись состояния, но не меняют множество возможных ходов.

4.3. Нейтральная игра *0

Нейтральный элемент

Игра *0 — элементарная проигрышная игра без ходов. Это терминальное состояние.

Свойство

A + *0 эквивалентна A.

Доказательство

В игре *0 нет ходов, поэтому в сумме A + *0 любой ход можно сделать только в компоненте A. Значит, игра ведёт себя ровно как A.

5. Лемма о симметричной стратегии: A + A проигрышна

Лемма

Если A — конечная ациклическая равноправная игра, то A + A является проигрышной игрой.

Интуиция

Две одинаковые копии игры взаимно уничтожаются. Что бы первый игрок ни сделал в одной копии, второй может повторить зеркальный ход в другой копии и снова восстановить симметрию.

Доказательство через зеркальную стратегию

Начальное состояние суммы имеет вид A + A. Оно симметрично: обе компоненты одинаковы.

Первый игрок обязан сделать ход в одной из компонент. Например, он переводит первую копию:

A + A → A' + A

Так как в исходной игре был допустим ход A → A', второй игрок может сделать точно такой же ход во второй копии:

A' + A → A' + A'

После ответа второго игрока состояние снова стало симметричным: две компоненты одинаковы.

Далее второй игрок повторяет эту стратегию: на каждый ход первого в одной копии отвечает тем же ходом в другой копии. Поскольку игра ациклична и конечна, бесконечно играть нельзя. В какой-то момент первый игрок не сможет сделать ход в симметричном состоянии T + T, где обе компоненты терминальны. Значит, первый игрок проигрывает, а A + A проигрышна.

Почему нужна ацикличность?

Если бы игра могла ходить по циклам, зеркальная стратегия могла бы не гарантировать завершения. В нашем классе игр граф ацикличен, поэтому каждый ход уменьшает некоторую «высоту» состояния, и партия обязательно закончится.

6. Эквивалентность игр по Гранди

Просто знать, выигрышна игра или проигрышна, недостаточно. Например, две выигрышные игры могут по-разному вести себя при сложении с третьей игрой. Поэтому вводят более сильное отношение — эквивалентность игр.

Эквивалентность

Игры A и B называются эквивалентными по Гранди, если для любой игры C значения сумм совпадают:

A ∼ B ⇔ ∀C: v(A + C) = v(B + C)

Иными словами, A и B неразличимы в любом контексте: какую бы дополнительную независимую игру C мы ни добавили, результат будет тем же.

6.1. Почему нельзя определять эквивалентность только как одинаковую выигрышность?

Пример

Игры *1 и *2 обе выигрышны: из каждой можно взять все камни и перейти в *0. Но они не эквивалентны.

Например, добавим *1:

  • *1 + *1 проигрышна по симметричной стратегии;
  • *2 + *1 выигрышна, потому что можно уменьшить кучку 2 до 1 и получить *1 + *1.

Значит, *1 и *2 дают разное поведение в суммах.

6.2. Лемма о совместимости эквивалентности со сложением

Лемма

Если A₁ ∼ A₂ и B₁ ∼ B₂, то A₁ + B₁ ∼ A₂ + B₂.

Доказательство

Нужно доказать, что для любой игры C:

v((A₁ + B₁) + C) = v((A₂ + B₂) + C)

Используем ассоциативность и коммутативность суммы игр.

Так как A₁ ∼ A₂, то для любой игры D выполняется:

v(A₁ + D) = v(A₂ + D)

Возьмём D = B₁ + C. Тогда:

v(A₁ + B₁ + C) = v(A₂ + B₁ + C)

Теперь используем B₁ ∼ B₂. Для любой игры E:

v(B₁ + E) = v(B₂ + E)

Возьмём E = A₂ + C. Тогда:

v(B₁ + A₂ + C) = v(B₂ + A₂ + C)

По коммутативности это то же самое, что:

v(A₂ + B₁ + C) = v(A₂ + B₂ + C)

Значит:

v(A₁ + B₁ + C) = v(A₂ + B₂ + C)

Для любого C. Следовательно, A₁+B₁ ∼ A₂+B₂.

6.3. Классы эквивалентности

Отношение ∼ разбивает все игры на классы эквивалентности. В одном классе лежат игры, которые ведут себя одинаково при добавлении любой другой игры.

Главная идея

Теорема Шпрага—Гранди говорит, что каждый такой класс можно представить одной ним-кучкой *g. Число g и называется функцией Гранди состояния.

7. Все проигрышные игры эквивалентны *0

Лемма

Если A — проигрышная игра, то для любой игры C:

v(A + C) = v(C)

Следовательно, A ∼ *0.

Доказательство через стратегию

Интуитивно проигрышная игра A не даёт активного преимущества. Если добавить её к любой игре C, результат должен быть таким же, как у C.

Случай 1: C выигрышна

Если C выигрышна, то в ней есть ход C → C', где C' проигрышна. Тогда в сумме A + C текущий игрок делает ход во второй компоненте:

A + C → A + C'

Нужно понять, почему A + C' проигрышна. И A, и C' проигрышны. В теории Гранди обе они эквивалентны *0, поэтому их сумма тоже эквивалентна *0. Если доказывать без уже готовой леммы, можно использовать индукцию по размеру игр.

Значит, из A+C есть ход в проигрышное состояние, поэтому A+C выигрышна, как и C.

Случай 2: C проигрышна

Если обе игры A и C проигрышны, то любой ход в одной из них переводит эту компоненту в выигрышное состояние. После этого соперник может пользоваться выигрышной стратегией в изменённой компоненте. Формально это доказывается индукцией по суммарной высоте состояний.

Следовательно, A+C проигрышна, как и C.

Аккуратность

В строгом курсе эту лемму удобно доказывать после введения функции Гранди: если игра проигрышна, её Grundy-число равно 0, а сумма с ней не меняет XOR. В лекционном стиле её часто дают раньше как мотивацию нейтрального класса *0.

8. Игра Ним

Одна кучка Нима

Есть кучка из i камней. За ход можно взять любое натуральное количество камней, то есть от 1 до i. Если камней 0, ходов нет, и позиция проигрышна.

Игра Ним из одной кучки размера i обозначается:

*i

Например:

  • *0 — пустая кучка, ходов нет;
  • *1 — можно перейти только в *0;
  • *2 — можно перейти в *1 или *0;
  • *i — можно перейти в любую *j, где 0 ≤ j < i.

8.1. Выигрышность одной кучки

Факт

*0 проигрышна. Любая *i при i > 0 выигрышна.

Доказательство

Если i = 0, ходов нет, значит позиция проигрышна. Если i > 0, текущий игрок может взять все i камней и перейти в *0, то есть в проигрышную позицию для соперника. Значит, позиция выигрышна.

9. Лемма: если i ≠ j, то *i ≁ *j

Лемма

Ним-кучки разных размеров неэквивалентны:

i ≠ j ⇒ *i ≁ *j
Доказательство

Пусть без ограничения общности i > j.

Рассмотрим добавочную игру C = *j. Тогда сравним:

*i + *j и *j + *j

Игра *j + *j проигрышна по лемме о симметричной стратегии: две одинаковые компоненты взаимно уничтожаются.

А игра *i + *j выигрышна. Почему? Первый игрок может в большей кучке *i оставить ровно j камней. То есть сделать ход:

*i + *j → *j + *j

Он переводит игру в проигрышную позицию для соперника. Значит, *i + *j выигрышна.

Мы нашли такую игру C = *j, что:

v(*i + C) ≠ v(*j + C)

Следовательно, *i и *j неэквивалентны.

Смысл

Каждая ним-кучка *i задаёт отдельный класс эквивалентности. Именно поэтому функция Гранди может принимать разные неотрицательные целые значения, а не только «win/lose».

10. Функция MEX

MEX

mex(S) — минимальное неотрицательное целое число, которого нет в множестве S.

Множество S mex(S) Почему
∅ 0 0 отсутствует.
{0} 1 0 есть, 1 отсутствует.
{0, 1, 2} 3 0, 1, 2 есть, 3 отсутствует.
{1, 2, 4} 0 0 отсутствует.
{0, 2, 3} 1 0 есть, 1 отсутствует.

MEX появляется потому, что игра *g имеет ходы ровно во все меньшие ним-кучки:

*g → *0, *1, ..., *(g - 1)

Поэтому, чтобы игра была эквивалентна *g, среди её достижимых состояний должны встречаться классы 0,1,...,g-1, но не должен встречаться класс g. Это ровно и выражает MEX.

11. I теорема Шпрага—Гранди

I теорема Шпрага—Гранди

Любая конечная ациклическая равноправная игра с полной информацией эквивалентна некоторой ним-кучке *g. Число g называется функцией Гранди этой игры.

Функция Гранди

Для состояния A его Grundy-число g(A) определяется так:

g(A) = mex({ g(B) | A → B })

То есть берём все состояния, достижимые одним ходом из A, смотрим их Grundy-числа, и берём MEX этого множества.

11.1. База: терминальное состояние

Если из A нет ходов, множество достижимых Grundy-чисел пусто:

{ g(B) | A → B } = ∅

Тогда:

g(A) = mex(∅) = 0

Это совпадает с тем, что терминальная игра эквивалентна *0.

11.2. Доказательство через индукцию по максимальному пути до терминала

Высота состояния

Высотой состояния назовём максимальную длину пути из него до терминального состояния. Так как граф ацикличен и конечен, высота корректно определена.

Доказательство I теоремы

Докажем индукцией по высоте состояния A, что оно эквивалентно *g(A), где:

g(A) = mex({ g(B) | A → B })

База

Если высота равна 0, состояние терминальное. Из него нет ходов, поэтому:

g(A) = mex(∅) = 0

Терминальная игра эквивалентна *0. База доказана.

Индукционный переход

Пусть для всех состояний меньшей высоты утверждение уже доказано. Рассмотрим состояние A. Все его потомки B имеют меньшую высоту, значит по предположению индукции каждый B эквивалентен *g(B).

Обозначим:

m = mex({ g(B) | A → B })

Нужно доказать, что A ∼ *m.

Ключевая проверка через добавление *m

Используем полезный критерий:

A ∼ *m ⇔ A + *m ∼ *0

То есть достаточно показать, что A + *m проигрышна.

Почему A + *m проигрышна?

Рассмотрим любой ход из A + *m. Он бывает двух типов.

Тип 1: ход в компоненте A.

Тогда игра переходит в B + *m, где A → B. По определению MEX число m не входит в множество достижимых Grundy-чисел. Значит, g(B) ≠ m для любого такого B.

По предположению индукции B ∼ *g(B). Поэтому:

B + *m ∼ *g(B) + *m

Так как g(B) ≠ m, сумма двух разных ним-кучек выигрышна: большую можно уменьшить до меньшей и получить две одинаковые кучки. Значит, ход типа 1 ведёт в выигрышное состояние.

Тип 2: ход в компоненте *m.

Из *m можно перейти в *t, где 0 ≤ t < m. Так как m = mex(S), все числа 0,1,...,m-1 присутствуют в множестве S = {g(B) | A → B}. Значит, найдётся ход A → B такой, что g(B)=t.

После хода типа 2 состояние стало A + *t. Для игрока, который теперь ходит, есть ответ: сделать в компоненте A ход в такое B, что g(B)=t. Тогда получится:

B + *t ∼ *t + *t ∼ *0

То есть ход типа 2 тоже ведёт в выигрышное для следующего игрока состояние.

Итак, любой ход из A + *m ведёт в выигрышное состояние. Значит, A + *m проигрышна. Поэтому A ∼ *m.

Замечание

В этом доказательстве неявно используется, что суммы ним-кучек с разными размерами выигрышны, а с одинаковыми — проигрышны. Для двух кучек это доказывается простой симметрией и ходом «уменьшить большую до меньшей».

12. II теорема Шпрага—Гранди

II теорема Шпрага—Гранди

Для ним-кучек выполняется:

*i + *j ∼ *(i xor j)

В лекции эта теорема дана без доказательства, но её смысл крайне важен для алгоритмов.

Более общий вывод:

*g₁ + *g₂ + ... + *gₙ ∼ *(g₁ xor g₂ xor ... xor gₙ)

То есть сумма независимых игр эквивалентна одной ним-кучке, размер которой равен XOR их Grundy-чисел.

Почему именно XOR?

XOR — это операция сложения в пространстве ним-значений. Она обладает теми свойствами, которые нужны сумме игр: коммутативность, ассоциативность, нейтральный элемент 0 и самоуничтожение x xor x = 0. Последнее соответствует лемме A + A ∼ *0.

12.1. Следствие для произвольной суммы игр

Если есть независимые игры A₁, A₂, ..., Aₙ, то сначала считаем их Grundy-числа:

g₁ = g(A₁), g₂ = g(A₂), ..., gₙ = g(Aₙ)

Тогда:

A₁ + A₂ + ... + Aₙ ∼ *(g₁ xor g₂ xor ... xor gₙ)

Поэтому позиция проигрышна тогда и только тогда, когда XOR всех Grundy-чисел равен 0.

Критерий проигрыша
A₁ + ... + Aₙ проигрышна ⇔ g(A₁) xor ... xor g(Aₙ) = 0

13. Оптимальная стратегия в сумме ним-игр

Пусть есть несколько компонент с Grundy-числами:

g₁, g₂, ..., gₙ

Обозначим их XOR:

X = g₁ xor g₂ xor ... xor gₙ

13.1. Если X = 0

Факт

Если общий XOR равен 0, то любой допустимый ход переводит игру в состояние с ненулевым XOR.

Доказательство

Один ход меняет только одну компоненту. Пусть g_i заменилось на g_i', причём g_i' ≠ g_i. Новый XOR равен:

X' = X xor g_i xor g_i'

Так как X = 0, получаем:

X' = g_i xor g_i'

А поскольку g_i ≠ g_i', их XOR ненулевой. Значит, X' ≠ 0.

13.2. Если X ≠ 0

Факт

Если общий XOR ненулевой, то существует ход, после которого общий XOR станет 0.

Доказательство для ним-куч

Пусть X ≠ 0. Возьмём старший бит, в котором у X стоит 1. Среди кучек найдётся кучка g_i, у которой в этом бите тоже стоит 1, иначе в XOR этот бит не мог бы стать 1.

Положим:

g_i' = g_i xor X

Из-за выбора старшего бита получаем g_i' < g_i. Значит, в ним-кучке *g_i можно сделать ход в *g_i', уменьшив кучку.

Новый общий XOR:

X' = X xor g_i xor g_i' = X xor g_i xor (g_i xor X) = 0

Значит, такой ход существует.

Практическая стратегия
  1. Посчитать общий XOR всех Grundy-чисел.
  2. Если XOR равен 0, позиция проигрышна при оптимальной игре.
  3. Если XOR ненулевой, найти компоненту g_i, для которой (g_i xor X) < g_i.
  4. Сделать ход в этой компоненте так, чтобы её новое Grundy-число стало g_i xor X.
Пример

Пусть кучки имеют размеры:

3, 4, 5

Считаем XOR:

3 xor 4 xor 5 = 2

XOR ненулевой, значит позиция выигрышна. Нужно найти кучку, которую можно уменьшить до g_i xor 2.

  • 3 xor 2 = 1 < 3 — подходит;
  • 4 xor 2 = 6 > 4 — не подходит;
  • 5 xor 2 = 7 > 5 — не подходит.

Уменьшаем кучку 3 до 1. Получаем:

1 xor 4 xor 5 = 0

14. Алгоритмы и код

14.1. Выигрышность в произвольном DAG

Если нужно определить только win/lose, достаточно DP по графу. 

vector<vector<int>> g;
vector<int> state; // -1 = unknown, 0 = lose, 1 = win

int winlose(int v) {
    if (state[v] != -1) {
        return state[v];
    }

    // Если ходов нет, позиция проигрышная.
    if (g[v].empty()) {
        return state[v] = 0;
    }

    // Если есть ход в проигрышную позицию, текущая выигрышная.
    for (int to : g[v]) {
        if (winlose(to) == 0) {
            return state[v] = 1;
        }
    }

    // Иначе все ходы ведут в выигрышные позиции.
    return state[v] = 0;
}

14.2. Grundy-функция в произвольном DAG

vector<vector<int>> g;
vector<int> grundy;

int calc_grundy(int v) {
    if (grundy[v] != -1) {
        return grundy[v];
    }

    vector<int> used;

    for (int to : g[v]) {
        used.push_back(calc_grundy(to));
    }

    sort(used.begin(), used.end());
    used.erase(unique(used.begin(), used.end()), used.end());

    int mex = 0;
    for (int x : used) {
        if (x == mex) {
            mex++;
        } else if (x > mex) {
            break;
        }
    }

    return grundy[v] = mex;
}

14.3. Более быстрый MEX через массив меток

Если состояний много, не хочется каждый раз сортировать. Можно использовать массив seen с временными метками. 

vector<vector<int>> g;
vector<int> grundy;
vector<int> seen;
int timer = 1;

int calc_grundy(int v) {
    if (grundy[v] != -1) {
        return grundy[v];
    }

    int cur_timer = timer++;

    for (int to : g[v]) {
        int x = calc_grundy(to);
        if (x >= (int)seen.size()) {
            seen.resize(x + 1, 0);
        }
        seen[x] = cur_timer;
    }

    int mex = 0;
    while (mex < (int)seen.size() && seen[mex] == cur_timer) {
        mex++;
    }

    return grundy[v] = mex;
}

14.4. Победа в сумме игр

int xorsum = 0;

for (int component : components) {
    xorsum ^= calc_grundy(component);
}

if (xorsum == 0) {
    cout << "LOSE\n";
} else {
    cout << "WIN\n";
}

14.5. Ним с несколькими кучками

int xorsum = 0;

for (int pile : piles) {
    xorsum ^= pile;
}

if (xorsum == 0) {
    cout << "Second player wins\n";
} else {
    cout << "First player wins\n";
}

15. Частые ошибки и тонкости

Ошибка 1

Думать, что любая выигрышная игра эквивалентна *1. Это неверно. Например, *1, *2, *3 все выигрышны, но попарно неэквивалентны.

Ошибка 2

Путать значение win/lose и Grundy-число. Win/lose — это только проверка, равно ли Grundy-число нулю.

A проигрышна ⇔ g(A) = 0 A выигрышна ⇔ g(A) ≠ 0
Ошибка 3

Считать, что Grundy суммы равен обычной сумме Grundy-чисел. На самом деле используется XOR:

g(A + B) = g(A) xor g(B)
Ошибка 4

Применять теорему Шпрага—Гранди к играм с циклами без дополнительной обработки. Теорема в этой форме работает для конечных ациклических игр.

Ошибка 5

Забывать, что ход в сумме игр делается только в одной компоненте. Нельзя за один ход изменить сразу несколько кучек или несколько независимых игр.

16. Короткая шпаргалка

Win/Lose

  • Нет ходов → проигрыш.
  • Есть ход в проигрыш → выигрыш.
  • Все ходы в выигрыш → проигрыш.

Сумма игр

  • Состояние: A+B.
  • За ход меняется ровно одна компонента.
  • *0 — нейтральная проигрышная игра.

Симметрия

  • A+A проигрышна.
  • Ответ: копировать ход соперника в другой компоненте.
  • Нужно, чтобы игра была конечной и ациклической.

Grundy

  • g(A)=mex({g(B) | A→B}).
  • g(A)=0 ⇔ позиция проигрышна.
  • g(A)≠0 ⇔ позиция выигрышна.

Ним

  • *i — кучка из i камней.
  • Из *i можно перейти в любую *j, где j<i.
  • *i + *j ∼ *(i xor j).

Сумма компонент

  • Считаем Grundy каждой компоненты.
  • Берём XOR всех значений.
  • XOR = 0 → проигрыш.
  • XOR ≠ 0 → выигрыш.
Главная формула: A₁ + A₂ + ... + Aₙ ∼ *(g(A₁) xor g(A₂) xor ... xor g(Aₙ))
Что надо уметь доказать на контрольной
  1. Критерий выигрышных и проигрышных состояний.
  2. Почему A+A проигрышна.
  3. Почему *i и *j неэквивалентны при i≠j.
  4. Почему Grundy-число считается через MEX.
  5. Почему сумма игр сводится к XOR Grundy-чисел.
  6. Почему XOR = 0 означает проигрыш, а XOR ≠ 0 означает выигрыш.
