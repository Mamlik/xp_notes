1. Задача восстановления по остаткам

Китайская теорема об остатках отвечает на вопрос:

$$ \text{Можно ли восстановить число } x \text{ по его остаткам при делении на разные модули?} $$

Например, известно:

$$ \begin{aligned} x &\equiv 2 \pmod 3,\\ x &\equiv 3 \pmod 5,\\ x &\equiv 2 \pmod 7. \end{aligned} $$

Нужно найти все целые $x$, удовлетворяющие системе.

Ответ в таких задачах обычно не одно число, а класс по модулю произведения модулей. Для примера выше произведение модулей равно $3 \cdot 5 \cdot 7 = 105$, и решение будет иметь вид:

$$ x \equiv 23 \pmod {105}. $$

Это значит, что подходят все числа:

$$ \ldots, -187, -82, 23, 128, 233, \ldots $$

2. Базовые определения

Определение: сравнение по модулю

Запись

$$a \equiv b \pmod m$$

означает, что $a$ и $b$ имеют одинаковый остаток при делении на $m$. Эквивалентно:

$$m \mid (a-b).$$

2.1. Свойства сравнений

Если $a \equiv b \pmod m$ и $c \equiv d \pmod m$, то:

$$ a+c \equiv b+d \pmod m, $$ $$ a-c \equiv b-d \pmod m, $$ $$ ac \equiv bd \pmod m. $$

Делить в сравнениях можно не всегда. Например, из

$$2x \equiv 2y \pmod 6$$

не следует $x \equiv y \pmod 6$. Можно делить только на число, обратимое по модулю.

Определение: обратный элемент по модулю

Число $a^{-1}$ называется обратным к $a$ по модулю $m$, если

$$a \cdot a^{-1} \equiv 1 \pmod m.$$

Такой обратный элемент существует тогда и только тогда, когда

$$\gcd(a,m)=1.$$
Почему обратный существует при gcd = 1

По расширенному алгоритму Евклида, если $\gcd(a,m)=1$, то существуют целые $u$ и $v$, такие что:

$$au + mv = 1.$$

Переходим к сравнению по модулю $m$:

$$au \equiv 1 \pmod m.$$

Значит, $u$ — обратный элемент к $a$ по модулю $m$.

3. Китайская теорема об остатках: основная формулировка

Теорема CRT для попарно взаимно простых модулей

Пусть модули

$$m_1, m_2, \ldots, m_n$$

попарно взаимно просты, то есть для любых $i \ne j$:

$$\gcd(m_i,m_j)=1.$$

Тогда для любых остатков

$$a_1, a_2, \ldots, a_n$$

система сравнений

$$ \begin{aligned} x &\equiv a_1 \pmod {m_1},\\ x &\equiv a_2 \pmod {m_2},\\ &\vdots\\ x &\equiv a_n \pmod {m_n} \end{aligned} $$

имеет решение. Более того, это решение единственно по модулю

$$M = m_1m_2\cdots m_n.$$

То есть существует ровно одно решение среди чисел:

$$0,1,2,\ldots,M-1.$$
Интуиция

Если модули попарно взаимно просты, то остатки по ним независимы. Остаток по модулю $3$ не навязывает скрытых ограничений на остаток по модулю $5$, и наоборот. Поэтому всего возможных наборов остатков:

$$m_1m_2\cdots m_n=M,$$

ровно столько же, сколько классов остатков по модулю $M$.

4. Доказательство единственности

Сначала докажем, что если решение существует, то оно единственно по модулю $M$.

Доказательство

Пусть $x$ и $y$ — два решения одной и той же системы:

$$x \equiv a_i \pmod {m_i}, \qquad y \equiv a_i \pmod {m_i}$$

для всех $i$.

Тогда для каждого $i$:

$$x-y \equiv 0 \pmod {m_i},$$

то есть

$$m_i \mid (x-y).$$

Так как модули попарно взаимно просты, из делимости на каждый $m_i$ следует делимость на их произведение:

$$M=m_1m_2\cdots m_n \mid (x-y).$$

Значит:

$$x \equiv y \pmod M.$$

То есть все решения отличаются на кратное $M$. Единственность по модулю $M$ доказана.

Почему из делимости на все m_i следует делимость на произведение

Если числа $m_1,\ldots,m_n$ попарно взаимно просты и каждое $m_i$ делит $N$, то произведение $M$ тоже делит $N$.

Для двух чисел это стандартный факт:

$$a \mid N,\quad b \mid N,\quad \gcd(a,b)=1 \quad \Longrightarrow \quad ab \mid N.$$

Доказательство: из $a \mid N$ получаем $N=ak$. Так как $b \mid N$, имеем $b \mid ak$. Но $\gcd(a,b)=1$, значит по лемме Евклида $b \mid k$. Тогда $k=bt$, и $N=abt$.

Для многих чисел применяем этот аргумент по индукции.

5. Доказательство существования для двух модулей

Начнём с системы из двух сравнений:

$$ \begin{aligned} x &\equiv a \pmod m,\\ x &\equiv b \pmod n, \end{aligned} \qquad \gcd(m,n)=1. $$
Конструктивное доказательство

Из первого сравнения следует:

$$x = a + mt$$

для некоторого целого $t$.

Подставим во второе сравнение:

$$a + mt \equiv b \pmod n.$$

Переносим $a$:

$$mt \equiv b-a \pmod n.$$

Так как $\gcd(m,n)=1$, у $m$ есть обратный элемент по модулю $n$. Умножаем на $m^{-1}$:

$$t \equiv (b-a)m^{-1} \pmod n.$$

Такой $t$ существует. Значит, существует и $x=a+mt$.

Пример

Решим:

$$ \begin{aligned} x &\equiv 2 \pmod 3,\\ x &\equiv 3 \pmod 5. \end{aligned} $$

Пишем $x=2+3t$. Подставляем во второе сравнение:

$$2+3t \equiv 3 \pmod 5,$$ $$3t \equiv 1 \pmod 5.$$

Обратный к $3$ по модулю $5$ равен $2$, потому что $3\cdot2=6\equiv1\pmod5$. Значит:

$$t \equiv 2 \pmod 5.$$

Берём $t=2$:

$$x=2+3\cdot2=8.$$

Ответ:

$$x \equiv 8 \pmod {15}.$$

6. Существование для многих модулей

Для многих модулей можно доказать существование двумя основными способами:

  1. индукцией, объединяя сравнения по одному;
  2. сразу выписать явную формулу решения.
Доказательство индукцией

Для одного модуля утверждение очевидно.

Пусть мы уже умеем решать систему из первых $k-1$ сравнений. Тогда существует решение:

$$x \equiv r \pmod {M_{k-1}},$$

где

$$M_{k-1}=m_1m_2\cdots m_{k-1}.$$

Теперь нужно совместить это с новым сравнением:

$$x \equiv a_k \pmod {m_k}.$$

Так как $m_k$ взаимно просто с каждым из $m_1,\ldots,m_{k-1}$, оно взаимно просто и с их произведением:

$$\gcd(M_{k-1},m_k)=1.$$

Значит, по случаю двух модулей система

$$ \begin{aligned} x &\equiv r \pmod {M_{k-1}},\\ x &\equiv a_k \pmod {m_k} \end{aligned} $$

имеет решение по модулю $M_{k-1}m_k$.

Индукция завершена.

7. Конструктивная формула CRT

Пусть дана система:

$$ x \equiv a_i \pmod {m_i}, \qquad i=1,2,\ldots,n, $$

где модули попарно взаимно просты. Обозначим:

$$M=m_1m_2\cdots m_n.$$

Для каждого $i$ положим:

$$M_i=\frac{M}{m_i}.$$

Так как $M_i$ содержит все модули кроме $m_i$, а модули попарно взаимно просты, то:

$$\gcd(M_i,m_i)=1.$$

Значит, существует обратный элемент к $M_i$ по модулю $m_i$. Обозначим его:

$$N_i \equiv M_i^{-1} \pmod {m_i}.$$
Явная формула решения
$$ x \equiv \sum_{i=1}^{n} a_i M_i N_i \pmod M. $$
Почему формула работает

Рассмотрим слагаемое $a_iM_iN_i$ по модулю $m_j$.

Если $j \ne i$, то $M_i$ содержит множитель $m_j$, поэтому:

$$M_i \equiv 0 \pmod {m_j}.$$

Значит:

$$a_iM_iN_i \equiv 0 \pmod {m_j}.$$

Если же $j=i$, то по определению $N_i$:

$$M_iN_i \equiv 1 \pmod {m_i}.$$

Поэтому:

$$a_iM_iN_i \equiv a_i \pmod {m_i}.$$

Итак, при рассмотрении всей суммы по модулю $m_j$ все слагаемые зануляются, кроме $j$-го, а оно даёт $a_j$:

$$x \equiv a_j \pmod {m_j}.$$

Формула доказана.

Подробный пример

Решим систему:

$$ \begin{aligned} x &\equiv 2 \pmod 3,\\ x &\equiv 3 \pmod 5,\\ x &\equiv 2 \pmod 7. \end{aligned} $$

Здесь:

$$M=3\cdot5\cdot7=105.$$

Считаем:

$i$$m_i$$a_i$$M_i=M/m_i$$N_i=M_i^{-1}\pmod {m_i}$
13235$35\equiv2\pmod3$, обратный к 2 — 2
25321$21\equiv1\pmod5$, обратный к 1 — 1
37215$15\equiv1\pmod7$, обратный к 1 — 1

Тогда:

$$ x \equiv 2\cdot35\cdot2 + 3\cdot21\cdot1 + 2\cdot15\cdot1 \pmod {105}. $$
$$x \equiv 140 + 63 + 30 = 233 \equiv 23 \pmod {105}.$$

Ответ:

$$x \equiv 23 \pmod {105}.$$
Практическое замечание

Эта формула красивая, но произведение $M$ может быть огромным. В задачах с большими модулями нужно либо работать с большими числами, либо использовать алгоритм Гарнера, либо восстанавливать ответ по дополнительному модулю.

8. Пошаговое объединение сравнений

Иногда удобнее объединять сравнения одно за другим.

Пусть уже есть:

$$x \equiv r \pmod M,$$

и нужно добавить:

$$x \equiv a \pmod m.$$

Если $\gcd(M,m)=1$, пишем:

$$x = r + Mt.$$

Подставляем:

$$r+Mt \equiv a \pmod m,$$ $$Mt \equiv a-r \pmod m.$$

Так как $M$ обратим по модулю $m$:

$$t \equiv (a-r)M^{-1} \pmod m.$$

После этого новое решение:

$$x \equiv r + Mt \pmod {Mm}.$$
Зачем нужен такой подход

Пошаговое объединение удобно, когда:

  • сравнения приходят онлайн;
  • модули не обязательно попарно взаимно просты;
  • нужно быстро проверить противоречивость системы;
  • ответ можно хранить в 128-битном типе или big integer.

9. Обобщение: модули не обязательно взаимно просты

Классическая CRT требует попарной взаимной простоты. Но системы сравнений можно решать и в общем случае.

Критерий совместности двух сравнений

Система

$$ \begin{aligned} x &\equiv a \pmod m,\\ x &\equiv b \pmod n \end{aligned} $$

имеет решение тогда и только тогда, когда

$$a \equiv b \pmod {\gcd(m,n)}.$$

Если решение существует, то оно единственно по модулю

$$\operatorname{lcm}(m,n).$$
Необходимость

Если $x$ удовлетворяет системе, то:

$$x-a \text{ делится на } m,$$ $$x-b \text{ делится на } n.$$

Пусть $d=\gcd(m,n)$. Тогда $d$ делит и $m$, и $n$, значит:

$$x \equiv a \pmod d,$$ $$x \equiv b \pmod d.$$

Следовательно:

$$a \equiv b \pmod d.$$
Достаточность

Пусть $d=\gcd(m,n)$ и $a\equiv b\pmod d$.

Ищем $x$ в виде:

$$x=a+mt.$$

Подставляем во второе сравнение:

$$a+mt \equiv b \pmod n,$$ $$mt \equiv b-a \pmod n.$$

Так как $d$ делит $m$, $n$ и $b-a$, можно разделить сравнение на $d$:

$$\frac{m}{d}t \equiv \frac{b-a}{d} \pmod {\frac{n}{d}}.$$

Теперь:

$$\gcd\left(\frac{m}{d},\frac{n}{d}\right)=1.$$

Значит, $m/d$ обратимо по модулю $n/d$, и $t$ существует. Следовательно, существует и $x$.

Пример совместной системы
$$ \begin{aligned} x &\equiv 2 \pmod 6,\\ x &\equiv 8 \pmod {10}. \end{aligned} $$

Здесь $\gcd(6,10)=2$, и $2\equiv8\pmod2$. Система совместна.

Решение будет единственно по модулю:

$$\operatorname{lcm}(6,10)=30.$$

Проверкой получаем:

$$x \equiv 8 \pmod {30}.$$
Пример несовместной системы
$$ \begin{aligned} x &\equiv 1 \pmod 4,\\ x &\equiv 2 \pmod 6. \end{aligned} $$

$\gcd(4,6)=2$, но $1\not\equiv2\pmod2$. Система несовместна.

10. Алгоритм Гарнера

Алгоритм Гарнера — это способ восстановить число по остаткам в специальной смешанной системе счисления.

Пусть модули $m_1,m_2,\ldots,m_n$ попарно взаимно просты, и известно:

$$x \equiv a_i \pmod {m_i}.$$

Вместо прямой формулы CRT Гарнер ищет $x$ в виде:

$$ x = c_1 + c_2m_1 + c_3m_1m_2 + c_4m_1m_2m_3 + \cdots + c_nm_1m_2\cdots m_{n-1}. $$

Коэффициенты удовлетворяют ограничениям:

$$0 \le c_i < m_i.$$
Смысл смешанной системы

В обычной десятичной системе число записывается как:

$$d_0 + d_1\cdot 10 + d_2\cdot 10^2 + \cdots.$$

В системе Гарнера основания разные:

$$1,\quad m_1,\quad m_1m_2,\quad m_1m_2m_3,\ldots$$

10.1. Как последовательно найти коэффициенты

Запишем разложение:

$$ x = c_1 + c_2m_1 + c_3m_1m_2 + \cdots + c_nm_1m_2\cdots m_{n-1}. $$

Первый коэффициент

По модулю $m_1$ все слагаемые кроме $c_1$ делятся на $m_1$. Поэтому:

$$c_1 \equiv a_1 \pmod {m_1}.$$

Берём:

$$c_1=a_1.$$

Второй коэффициент

По модулю $m_2$:

$$c_1 + c_2m_1 \equiv a_2 \pmod {m_2}.$$

Значит:

$$c_2m_1 \equiv a_2-c_1 \pmod {m_2}.$$

Так как $m_1$ обратим по модулю $m_2$:

$$c_2 \equiv (a_2-c_1)m_1^{-1} \pmod {m_2}.$$

Третий коэффициент

По модулю $m_3$:

$$c_1+c_2m_1+c_3m_1m_2 \equiv a_3 \pmod {m_3}.$$

Отсюда:

$$c_3 \equiv \left(a_3-c_1-c_2m_1\right)(m_1m_2)^{-1} \pmod {m_3}.$$

Общий вид

Пусть уже найдены $c_1,\ldots,c_{i-1}$. Тогда:

$$ c_i \equiv \left(a_i - \sum_{j=1}^{i-1} c_jm_1m_2\cdots m_{j-1}\right) \left(m_1m_2\cdots m_{i-1}\right)^{-1} \pmod {m_i}. $$

Здесь пустое произведение $m_1m_2\cdots m_0$ считается равным $1$.

Проблема прямого вычисления

Если прямо считать произведения $m_1m_2\cdots m_{i-1}$, они могут быстро переполнить тип. Поэтому часто используют более аккуратную форму Гарнера, которая преобразует остатки постепенно и хранит всё по модулям.

10.2. Практическая форма Гарнера

Идея: пусть текущие значения $x_i$ сначала равны исходным остаткам $a_i$. Затем мы последовательно превращаем их в коэффициенты $c_i$.

Для каждого $i$ после обработки предыдущих коэффициентов значение $x_i$ будет равно будущему $c_i$ по модулю $m_i$.

Алгоритм 
for i = 1..n:
    c[i] = a[i]

for i = 1..n:
    c[i] = c[i] mod m[i]
    for j = i+1..n:
        c[j] = (c[j] - c[i]) * inverse(m[i], m[j]) mod m[j]

После этого массив $c$ содержит коэффициенты смешанного разложения:

$$ x = c_1 + c_2m_1 + c_3m_1m_2 + \cdots + c_nm_1m_2\cdots m_{n-1}. $$

10.3. Пример Гарнера

Решим систему:

$$ \begin{aligned} x &\equiv 2 \pmod 3,\\ x &\equiv 3 \pmod 5,\\ x &\equiv 2 \pmod 7. \end{aligned} $$

Ищем:

$$x=c_1+c_2\cdot3+c_3\cdot3\cdot5.$$

Первый коэффициент:

$$c_1=2.$$

Второй:

$$2+3c_2\equiv3\pmod5,$$ $$3c_2\equiv1\pmod5,$$ $$c_2\equiv2\pmod5.$$

Значит $c_2=2$.

Третий:

$$2+2\cdot3+15c_3\equiv2\pmod7.$$

Левая часть без $c_3$ равна $8$, значит:

$$8+15c_3\equiv2\pmod7.$$

Сокращаем по модулю $7$:

$$1+c_3\equiv2\pmod7,$$ $$c_3\equiv1\pmod7.$$

Итого:

$$x=2+2\cdot3+1\cdot15=23.$$

11. Доказательство алгоритма Гарнера

Шаг 1: единственность смешанного представления

Покажем, что каждое число $0\le x

$$ x=c_1+c_2m_1+c_3m_1m_2+\cdots+c_nm_1m_2\cdots m_{n-1}, $$

где $0\le c_i

Это аналог обычной записи числа в позиционной системе счисления, только основания меняются. Можно строить коэффициенты делением:

$$c_1 = x \bmod m_1,$$ $$\frac{x-c_1}{m_1} = c_2 + c_3m_2 + \cdots.$$

Потом берём остаток по $m_2$, получаем $c_2$, и так далее.

Единственность следует из того, что на каждом шаге остаток по очередному модулю однозначно задаёт очередную цифру.

Шаг 2: почему коэффициенты можно находить последовательно

Подставим смешанное разложение в сравнение по модулю $m_i$:

$$ c_1+c_2m_1+\cdots+c_i m_1m_2\cdots m_{i-1}+\cdots \equiv a_i \pmod {m_i}. $$

Все слагаемые после $c_i$ содержат множитель $m_i$, потому что в них есть произведение $m_1m_2\cdots m_i$. Поэтому они равны нулю по модулю $m_i$.

Остаётся:

$$ c_1+c_2m_1+\cdots+c_i m_1m_2\cdots m_{i-1} \equiv a_i \pmod {m_i}. $$

Коэффициенты $c_1,\ldots,c_{i-1}$ уже известны. Значит, это линейное сравнение относительно $c_i$:

$$ c_i \cdot (m_1m_2\cdots m_{i-1}) \equiv a_i - \sum_{j=1}^{i-1} c_jm_1m_2\cdots m_{j-1} \pmod {m_i}. $$

Так как модули попарно взаимно просты, произведение $m_1m_2\cdots m_{i-1}$ обратимо по модулю $m_i$. Поэтому $c_i$ находится однозначно.

Шаг 3: почему практическая форма с обновлением остатков работает

Сначала у нас есть система:

$$x \equiv a_j \pmod {m_j}.$$

Пусть мы нашли $c_1$. Тогда:

$$x=c_1+m_1y.$$

Для каждого $j>1$ подставляем в сравнение по модулю $m_j$:

$$c_1+m_1y\equiv a_j\pmod {m_j}.$$

Отсюда:

$$y\equiv (a_j-c_1)m_1^{-1}\pmod {m_j}.$$

Именно это делает обновление:

$$a_j \leftarrow (a_j-c_1)m_1^{-1}\pmod {m_j}.$$

После этого мы решаем такую же задачу, но уже для $y$ и модулей $m_2,\ldots,m_n$. Повторяя процесс, получаем все коэффициенты.

12. Реализации на C++

12.1. Расширенный алгоритм Евклида и обратный элемент

using long long = long long;
long long extgcd(long long a, long long b, long long &x, long long &y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }
    long long x1, y1;
    long long g = extgcd(b, a % b, x1, y1);
    x = y1;
    y = x1 - (a / b) * y1;
    return g;
}

long long mod_norm(long long x, long long mod) {
    x %= mod;
    if (x < 0) x += mod;
    return x;
}

long long inv_mod(long long a, long long mod) {
    long long x, y;
    long long g = extgcd(a, mod, x, y);
    if (g != 1) {
        throw runtime_error("inverse does not exist");
    }
    return mod_norm(x, mod);
}

12.2. CRT для попарно взаимно простых модулей через формулу

long long crt_coprime_formula(const vector<long long>& a,
                              const vector<long long>& m) {
    int n = (int)a.size();
    __int128 M = 1;
    for (long long mod : m) {
        M *= mod;
    }

    __int128 ans = 0;

    for (int i = 0; i < n; i++) {
        __int128 Mi = M / m[i];
        long long Mi_mod = (long long)(Mi % m[i]);
        long long Ni = inv_mod(Mi_mod, m[i]);
        ans = (ans + (__int128)a[i] * Mi % M * Ni) % M;
    }

    long long res = (long long)(ans % M);
    if (res < 0) res += (long long)M;
    return res;
}
Осторожно

Этот код использует __int128, но если произведение модулей не помещается даже в 128 бит, нужен big integer или другой подход.

12.3. Общее объединение двух сравнений

Функция возвращает пару $(r, M)$, означающую:

$$x \equiv r \pmod M.$$

Если система несовместна, возвращает false.

struct Congruence {
    long long r;
    long long m;
};

bool merge_congruence(Congruence a, Congruence b, Congruence &res) {
    // x = a.r mod a.m
    // x = b.r mod b.m
    long long r1 = a.r, m1 = a.m;
    long long r2 = b.r, m2 = b.m;

    long long x, y;
    long long g = extgcd(m1, m2, x, y);

    if ((r2 - r1) % g != 0) {
        return false;
    }

    long long m2_g = m2 / g;

    // t = ((r2-r1)/g) * inverse(m1/g mod m2/g)
    __int128 t = (__int128)(r2 - r1) / g * x;
    t = t % m2_g;
    if (t < 0) t += m2_g;

    __int128 lcm = (__int128)m1 / g * m2;
    __int128 rr = (__int128)r1 + (__int128)m1 * t;
    rr %= lcm;
    if (rr < 0) rr += lcm;

    res.r = (long long)rr;
    res.m = (long long)lcm;
    return true;
}

12.4. Алгоритм Гарнера

Базовая версия для попарно взаимно простых модулей и результата, помещающегося в 64 бита.

long long garner(vector<long long> a, vector<long long> m) {
    int n = (int)a.size();
    vector<long long> c = a;

    for (int i = 0; i < n; i++) {
        c[i] = mod_norm(c[i], m[i]);
        for (int j = i + 1; j < n; j++) {
            long long inv = inv_mod(m[i] % m[j], m[j]);
            c[j] = mod_norm((__int128)(c[j] - c[i]) * inv, m[j]);
        }
    }

    __int128 x = 0;
    __int128 mult = 1;
    for (int i = 0; i < n; i++) {
        x += (__int128)c[i] * mult;
        mult *= m[i];
    }

    return (long long)x;
}

12.5. Гарнер для восстановления по дополнительному модулю

Частая олимпиадная задача: число $x$ огромное, но нужно найти его по модулю $P$.

Тогда добавляют $P$ как дополнительный модуль в конец и запускают Гарнера. Но это корректно напрямую только если $P$ взаимно прост с исходными модулями.

long long garner_mod(vector<long long> a, vector<long long> m, long long MOD) {
    int n = (int)a.size();

    // Добавляем фиктивное сравнение x = 0 mod MOD.
    // Его коэффициент в смешанной записи даст x mod MOD.
    a.push_back(0);
    m.push_back(MOD);
    n++;

    vector<long long> c = a;

    for (int i = 0; i < n; i++) {
        c[i] = mod_norm(c[i], m[i]);
        for (int j = i + 1; j < n; j++) {
            long long inv = inv_mod(m[i] % m[j], m[j]);
            c[j] = mod_norm((__int128)(c[j] - c[i]) * inv, m[j]);
        }
    }

    // Восстанавливаем только по MOD.
    long long result = 0;
    long long mult = 1 % MOD;
    for (int i = 0; i + 1 < n; i++) {
        result = (result + (__int128)c[i] * mult) % MOD;
        mult = (__int128)mult * (m[i] % MOD) % MOD;
    }
    return result;
}

13. Важные применения CRT и Гарнера

1. Работа с большими числами через остатки

Можно хранить огромное число не в десятичной записи, а как набор остатков:

$$x \mapsto (x\bmod m_1, x\bmod m_2, \ldots, x\bmod m_n).$$

Сложение и умножение выполняются покомпонентно:

$$(x+y)\bmod m_i = ((x\bmod m_i)+(y\bmod m_i))\bmod m_i,$$ $$(xy)\bmod m_i = ((x\bmod m_i)(y\bmod m_i))\bmod m_i.$$

CRT гарантирует, что если произведение модулей достаточно велико, то число можно восстановить однозначно.

2. Ускорение умножения больших чисел и полиномов

В FFT/NTT иногда считают свёртку по нескольким удобным простым модулям, а потом восстанавливают точные коэффициенты через CRT.

Например, если один модуль не даёт достаточного диапазона, используют несколько NTT-friendly простых модулей и собирают результат обратно.

3. RSA и криптография

В RSA секретный модуль имеет вид:

$$N=pq.$$

Вместо вычисления степени по модулю $N$ можно отдельно считать по модулям $p$ и $q$, а затем восстановить ответ по CRT. Это значительно быстрее, потому что операции с числами примерно вдвое меньшей длины дешевле.

4. Системы периодов и расписания

Задачи вида:

$$t \equiv a \pmod m, \qquad t \equiv b \pmod n$$

естественно возникают при совмещении циклических событий: расписаний, периодических процессов, календарных условий.

5. Хеширование и проверка равенства больших значений

Иногда значение проверяют по нескольким модулям. Если остатки совпадают по нескольким взаимно простым модулям, то разность делится на их произведение. Чем больше произведение, тем сильнее гарантия.

6. Диофантовы уравнения и линейные сравнения

CRT тесно связана с решением линейных сравнений и систем ограничений на целые числа. Обобщённая CRT даёт критерий совместности систем с непопарно взаимно простыми модулями.

7. Абстрактная алгебра

CRT имеет кольцевую формулировку:

$$ \mathbb Z / M\mathbb Z \cong \mathbb Z / m_1\mathbb Z \times \cdots \times \mathbb Z / m_n\mathbb Z $$

при попарно взаимно простых $m_i$ и $M=m_1m_2\cdots m_n$.

Это означает, что арифметика по модулю $M$ эквивалентна покомпонентной арифметике по модулям $m_i$.

14. Типичные ошибки

Ошибка 1: забыть условие взаимной простоты

Формула CRT через $M_i^{-1}$ работает только если $M_i$ обратим по модулю $m_i$. Для этого нужны попарно взаимно простые модули.

Ошибка 2: считать, что решение всегда существует

Для непопарно взаимно простых модулей система может быть противоречивой. Нужно проверять:

$$a_i \equiv a_j \pmod {\gcd(m_i,m_j)}.$$
Ошибка 3: отрицательные остатки

В C++ операция % может вернуть отрицательное число. Лучше нормализовать:

x %= mod;
if (x < 0) x += mod;
Ошибка 4: переполнение произведения модулей

Даже если каждый модуль помещается в long long, их произведение может не помещаться. Используйте __int128, big integer или Гарнера по дополнительному модулю.

Ошибка 5: неверный порядок в Гарнере

Коэффициенты Гарнера зависят от порядка модулей. Это не ошибка: смешанная система счисления строится в выбранном порядке. Итоговое число всё равно будет тем же по модулю произведения, если всё посчитано корректно.

15. Краткая шпаргалка

CRT

Если $m_i$ попарно взаимно просты, то система

$$x\equiv a_i\pmod {m_i}$$

имеет единственное решение по модулю $M=m_1\cdots m_n$.

Явная формула
$$M_i=\frac{M}{m_i},\qquad N_i\equiv M_i^{-1}\pmod {m_i}$$ $$x\equiv\sum_i a_iM_iN_i\pmod M$$
Два не взаимно простых модуля

Система

$$x\equiv a\pmod m,\qquad x\equiv b\pmod n$$

совместна тогда и только тогда, когда

$$a\equiv b\pmod {\gcd(m,n)}.$$
Гарнер

Ищем:

$$x=c_1+c_2m_1+c_3m_1m_2+\cdots$$

Коэффициенты находятся последовательно по модулям.
